# 合并发送图片展示实现说明

这份文档只讲一种聊天界面的多图展示方式：同一条消息里有多张图片时，把它们显示成一摞可滑动切换的图片卡片。效果类似“几张图片叠在一起，当前图在最前，滑一下切到下一张”。

## 目标效果

1. 一条消息里只有 1 张图片时，按普通单图展示。
2. 一条消息里有 2 张及以上图片时，合并成一个叠图卡片。
3. 叠图卡片只显示当前图片，后面的图片露出边缘，形成一摞图的感觉。
4. 横向滑动切换图片，一次只切换一张。
5. 点击卡片时，只查看当前最前面的图片。
6. 切换时，当前图退到后面，下一张图从后面滑到前面。
7. 切换结束后，卡片不能再弹一下、重排一下、抖一下。
8. 如果聊天头像在旁边，靠头像一侧的露边要更窄，避免图片叠层遮住头像。

## 基本数据结构

每张图片至少需要有稳定 id 和缩略图地址：

```ts
type ImageItem = {
  id: string;
  src: string;
  alt?: string;
};
```

组件入口：

```ts
function MergedImageStack({
  images,
  align = "left",
  onPreview,
}: {
  images: ImageItem[];
  align?: "left" | "right";
  onPreview?: (image: ImageItem) => void;
}) {
  if (images.length < 2) return null;
}
```

`align` 用来判断这条消息在左侧还是右侧，从而决定哪一边靠近头像。

## 核心状态

不要只用一个 `activeIndex` 然后在每次滑动时动态重排所有图片。这样很容易在滑动结束后又重新排一次，造成“换完还在动”。

推荐使用稳定状态 + 临时运动状态：

```ts
type StackMotion = {
  phase: "dragging" | "settling";
  baseIndex: number;
  direction: 1 | -1;
  targetIndex: number | null;
  progress: number;
  accepted: boolean;
};

const [activeIndex, setActiveIndex] = useState(0);
const [motion, setMotion] = useState<StackMotion | null>(null);
```

含义：

- `activeIndex`：当前稳定状态下最前面的图片索引。
- `motion`：正在拖动或收尾动画时的临时状态。
- `baseIndex`：本次拖动开始时的索引，拖动过程中固定不变。
- `direction`：`1` 表示切到下一张，`-1` 表示切到上一张。
- `targetIndex`：本次要切到的相邻图片索引。
- `progress`：切换进度，范围 0 到 1。
- `accepted`：松手后是否确认切换。

同时用 ref 保存最新 motion，避免动画回调拿到旧状态：

```ts
const motionRef = useRef<StackMotion | null>(null);

function setStackMotion(next: StackMotion | null) {
  motionRef.current = next;
  setMotion(next);
}
```

## 卡片姿态

每张图片的位置用一个姿态对象描述：

```ts
type CardPose = {
  x: number;
  y: number;
  scale: number;
  rotate: number;
  opacity: number;
};
```

`offset` 表示某张图相对当前前景图的位置：

- `0`：当前最前面的图。
- `1`：右侧/下一张方向的第一张背景图。
- `-1`：左侧/上一张方向的第一张背景图。
- `2`、`-2`：更深一层。

姿态计算：

```ts
function stackPose(offset: number, align: "left" | "right"): CardPose & { zIndex: number } {
  const rawDepth = Math.abs(offset);
  if (rawDepth < 0.001) {
    return { x: 0, y: 0, scale: 1, rotate: 0, opacity: 1, zIndex: 20 };
  }

  const side = offset > 0 ? 1 : -1;
  const towardAvatar = side === (align === "right" ? 1 : -1);
  const depth = Math.min(rawDepth, 3);
  const firstLayer = Math.min(depth, 1);
  const extraDepth = Math.max(depth - 1, 0);

  const xSpread = towardAvatar
    ? 8 * firstLayer + extraDepth * 4
    : 18 * firstLayer + extraDepth * 12;

  const rotateSpread = towardAvatar
    ? 0.45 * firstLayer + extraDepth * 0.2
    : 1.15 * firstLayer + extraDepth * 0.5;

  const y = 6 * firstLayer + extraDepth * 5;
  const scale = 1 - depth * 0.04;
  const opacity = Math.max(
    towardAvatar ? 0.34 : 0.44,
    1 - depth * (towardAvatar ? 0.22 : 0.18),
  );
  const zIndex = Math.max(1, 10 - Math.ceil(depth));

  return {
    x: side * xSpread,
    y,
    scale,
    rotate: side * rotateSpread,
    opacity,
    zIndex,
  };
}
```

这里的关键是 `towardAvatar`：靠头像那边露得少一点，远离头像那边正常展开。

## 三层结构

渲染时拆成三类层：

1. `background`：背景图层，不是当前图，也不是目标图。
2. `target`：正在切换进来的目标图。
3. `current`：正在切换出去的当前图。

结构：

```tsx
<button className="merged-image-stack">
  {backgroundEntries.map(renderBackground)}
  {targetItem && motion ? renderTarget() : null}
  {renderCurrent()}
</button>
```

所有图片层都绝对定位在同一个容器里：

```tsx
<span
  className="image-card"
  style={{
    transform: `translate3d(${pose.x}px, ${pose.y}px, 0) scale(${pose.scale}) rotate(${pose.rotate}deg)`,
    opacity: pose.opacity,
    zIndex,
    transition,
  }}
>
  <img src={image.src} alt={image.alt || ""} draggable={false} />
</span>
```

CSS：

```css
.merged-image-stack {
  position: relative;
  width: 162px;
  height: 216px;
  overflow: visible;
  touch-action: pan-y;
  perspective: 800px;
  transform-style: preserve-3d;
}

.image-card {
```

## 避免结束后重排

这是最重要的一点。

错误做法：滑动时背景层固定不动，松手后更新 `activeIndex`，再让所有背景图按新索引重算位置。这样会导致切换结束后后面的图又动一下。

正确做法：滑动过程中就让背景层跟着移动到最终位置。

```ts
const baseIndex = motion?.baseIndex ?? activeIndex;
const motionProgress = motion
  ? motion.phase === "dragging"
    ? Math.pow(motion.progress, 0.92)
    : motion.progress
  : 0;

const stackShift = motion?.targetIndex != null
  ? motion.direction * motionProgress
  : 0;

const stackAnchorIndex = baseIndex + stackShift;

const backgroundEntries = images
  .map((image, index) => ({
    image,
    index,
    offset: index - stackAnchorIndex,
  }))
  .filter(({ index, offset }) => (
    index !== baseIndex &&
    index !== motion?.targetIndex &&
    Math.abs(offset) <= 3.2
  ));
```

当 `progress` 到 1 时，背景层已经在最终位置了。之后清掉 `motion`，不会再触发肉眼可见的重排。

## 当前图和目标图动画

插值函数：

```ts
function mixPose(from: CardPose, to: CardPose, progress: number): CardPose {
  const p = Math.max(0, Math.min(progress, 1));
  return {
    x: from.x + (to.x - from.x) * p,
    y: from.y + (to.y - from.y) * p,
    scale: from.scale + (to.scale - from.scale) * p,
    rotate: from.rotate + (to.rotate - from.rotate) * p,
    opacity: from.opacity + (to.opacity - from.opacity) * p,
  };
}
```

当前图：从正中退到目标方向的反侧背景位。

```ts
const frontPose = { x: 0, y: 0, scale: 1, rotate: 0, opacity: 1 };
const currentRestPose = motion?.targetIndex != null
  ? stackPose(-motion.direction, align)
  : null;

const currentPose = motion
  ? mixPose(frontPose, currentRestPose || frontPose, motionProgress)
  : frontPose;
```

目标图：从目标方向的背景位进入正中。

```ts
const targetStartPose = motion?.targetIndex != null
  ? stackPose(motion.direction, align)
  : null;

const targetPose = motion && targetItem
  ? mixPose(targetStartPose || frontPose, frontPose, motionProgress)
  : frontPose;
```

这样就会形成“前图退后，后图盖上来”的翻卡片感。

## z-index 翻层

不要让 z-index 跟着手指的小抖动来回变化。用一个固定翻层点：

```ts
const layerFlipProgress = 0.48;

const targetLayerZIndex = targetStartPose && motionProgress < layerFlipProgress
  ? targetStartPose.zIndex
  : 24;

const currentLayerZIndex = currentRestPose && motionProgress >= layerFlipProgress
  ? currentRestPose.zIndex
  : 24;
```

含义：

- 目标图前半段还在后面，后半段升到前面。
- 当前图前半段还在前面，后半段退到后面。
- 翻层点固定，避免两张图来回抢前景。

## 滑动逻辑

参数建议：

```ts
const swipeDistance = 136;
const dragActivationDistance = 14;
const commitDistance = 42;
```

### pointerdown

```ts
function handlePointerDown(event: React.PointerEvent) {
  if (motionRef.current?.phase === "settling") return;

  dragStartX.current = event.clientX;
  dragBaseIndexRef.current = activeIndex;
  dragDirectionRef.current = null;
  suppressClickRef.current = false;
  setStackMotion(null);

  event.currentTarget.setPointerCapture?.(event.pointerId);
}
```

### pointermove

```ts
function handlePointerMove(event: React.PointerEvent) {
  const startX = dragStartX.current;
  if (startX == null) return;

  const deltaX = event.clientX - startX;
  if (Math.abs(deltaX) > 6) suppressClickRef.current = true;
  if (Math.abs(deltaX) <= dragActivationDistance) return;

  const direction = dragDirectionRef.current ?? (deltaX < 0 ? 1 : -1);
  dragDirectionRef.current = direction;

  const base = dragBaseIndexRef.current;
  const directionalDelta = direction === 1 ? -deltaX : deltaX;
  const effectiveDistance = Math.max(0, directionalDelta - dragActivationDistance);
  const nextIndex = base + direction;
  const hasTarget = nextIndex >= 0 && nextIndex < images.length;
  const progress = hasTarget
    ? Math.min(effectiveDistance / swipeDistance, 1)
    : Math.min(effectiveDistance / swipeDistance, 0.18);

  setStackMotion({
    phase: "dragging",
    baseIndex: base,
    direction,
    targetIndex: hasTarget ? nextIndex : null,
    progress,
    accepted: false,
  });
}
```

### pointerup

```ts
function handlePointerUp(event: React.PointerEvent) {
  const startX = dragStartX.current;
  dragStartX.current = null;

  if (startX == null) return;

  const lockedDirection = dragDirectionRef.current;
  dragDirectionRef.current = null;

  const deltaX = event.clientX - startX;
  const direction = lockedDirection ?? (deltaX < 0 ? 1 : -1);
  const base = dragBaseIndexRef.current;
  const directionalDelta = direction === 1 ? -deltaX : deltaX;
  const effectiveDistance = directionalDelta - dragActivationDistance;
  const nextIndex = base + direction;
  const hasTarget = nextIndex >= 0 && nextIndex < images.length;
  const accepted = effectiveDistance >= commitDistance && hasTarget;

  suppressClickRef.current = true;

  setStackMotion({
    phase: "settling",
    baseIndex: base,
    direction,
    targetIndex: hasTarget ? nextIndex : null,
    progress: accepted ? 1 : 0,
    accepted,
  });

  queueSettleFallback();
}
```

## 动画结束

只让当前图层监听 `transitionend`，不要让背景层和目标层都触发结束逻辑。

```ts
function finishSettling() {
  const current = motionRef.current;
  if (!current || current.phase !== "settling") return;

  clearSettleTimer();

  if (current.accepted && current.targetIndex != null) {
    setActiveIndex(current.targetIndex);
  }

  setStackMotion(null);
}

function handleTransitionEnd(event: React.TransitionEvent) {
  if (event.target !== event.currentTarget) return;
  if (event.propertyName !== "transform") return;
  finishSettling();
}
```

加 fallback，防止某些 WebView 丢 `transitionend`：

```ts
function queueSettleFallback() {
  clearSettleTimer();
  settleTimerRef.current = window.setTimeout(finishSettling, 320);
}
```

## 点击预览

滑动后会触发 click，所以要吞掉一次：

```ts
function handleClick() {
  if (suppressClickRef.current) {
    suppressClickRef.current = false;
    return;
  }

  onPreview?.(images[activeIndex]);
}
```

## 需要避免的坑

1. 不要滑完后才重排背景图，要在滑动过程中用 `stackAnchorIndex` 让背景层移动到最终位置。
2. 不要用拖动距离直接计算要跳到第几张，一次滑动只允许切相邻一张。
3. 不要让所有卡片都动态排序，拆成 current、target、background 三类层。
4. 不要让 z-index 随手指抖动反复变化，用固定翻层点。
5. 不要让内部图片层接管触摸事件，事件统一放在外层。
6. 不要忘记吞掉滑动后的 click，否则会误打开预览。
7. 不要让靠头像一侧露太多，图片多时会遮住头像。
