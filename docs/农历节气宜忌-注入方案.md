# 时间与农历注入（已实现）

## 时区

全部按 **北京时间（UTC+8）**：今日日期、当前大概时段、兜底具体时间、农历节气宜忌的展示均用北京时间。

## 当前逻辑

- **常驻注入**（每轮都有）：
  - **今日**：`YYYY-MM-DD`（北京时间）
  - **当前大概**：`早上` / `上午` / `中午` / `下午` / `傍晚` / `晚上` / `深夜`（按北京时间小时映射）
  - **system 固定一句**：「如需知道当前几点，可使用 get_time_info 工具」

- **兜底具体时间**（仅当**渡的上一轮回复**是问句且含「几点/时间/现在」时，本轮注入）：
  - 追加一行：`当前时间：HH:mm`（北京时间）

- **农历节气宜忌**（仅当**渡的上一轮回复**含「农历/节气/宜忌/黄历」时，本轮注入）：
  - 追加一行：`农历x月x日 节气：… 宜忌：…`（农历用 zhdate；节气/宜忌当前为占位「见黄历」）

## 配置（环境变量）

- `ASSISTANT_TIME_KEYWORDS`：渡的上一轮回复含这些词且为问句时才注入具体时间，逗号分隔。默认：`几点,时间,现在`。
- `ASSISTANT_LUNAR_KEYWORDS`：渡的上一轮回复含任一词则注入农历节气宜忌，逗号分隔。默认：`农历,节气,宜忌,黄历`。

## 实现位置

- **时间与农历**：`utils/time_aware.py`（全部北京时间：`get_date_only`、`get_time_period`、`get_exact_time`、`get_lunar_and_terms`）。
- **注入与触发**：`pipeline/pipeline.py` 的 `step_inject_summary`：用 `_last_assistant_text` 取渡的上一轮回复，`_is_question_like` 判问句，再按关键词决定是否追加「当前时间」和「农历节气宜忌」。
- **农历依赖**：`zhdate`（见 `requirements.txt`）；未安装时农历段显示占位。
- **节气 / 宜忌**：当前为占位，后续可在 `get_lunar_and_terms` 内接表或 API。
