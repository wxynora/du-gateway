# 记忆系统改进方案（需求备忘）

本文记录待演进的能力与产品取向，**实现以仓库代码为准**。

---

## 一、基础改进：动态层 = 中期记忆（与窗口总结分工）

窗口总结**只承担**「当前对话不断档、最基础不失忆」，**不**改为情感叙事或复杂策略的主载体。**中期记忆**由动态层承担，需求如下。

### 1.1 定位

- **动态层**：跨轮、可检索、可演化的**中期记忆**（离散短句为主）。
- **窗口总结**：同窗口内**短期连贯**，不与动态层抢叙事职责。

### 1.2 记什么（准入）

- **优先**：关系与情感线索、约定与承诺、稳定偏好与底线、会反复影响后续互动的背景。
- **其次**：近期仍会被提起的具体事实（宜短、宜合并）。
- **避免**：流水账、一次性可查信息、语义重复却不 merge 的堆叠。

### 1.3 形态与演化

- 单条宜**短而准**；同一主题优先 **merge** 收敛，避免多条平行变体。
- **importance** 表示**中期保留价值**（关系 / 情感 / 约定偏高），而非「这句话写得好不好」。
- 新信息与旧条冲突时，应**更新或合并**，减少过期状态长期占位。

### 1.4 边缘淘汰（仅动态层）

- 对**权重已很低且久未更新**的记忆，从动态层落盘与向量索引中清理，**不碰**核心缓存 pending。
- 与「7 天内是否仍参与注入」的规则**解耦**：注入有效期管「进不进本轮 prompt」，边缘淘汰管「删不删池子里沉积条」。

### 1.5 检索与注入

- 召回与排序偏向**与当前话题相关且对互动有影响**的条目；**重质不重量**，不追求条数多。

---

## 二、进阶机制（在基础之上叠加）

### 2.1 置信度 / 来源类型（影响排序）

- 为动态层条目维护**来源类元信息**（如用户直说、推断、合并产物、多轮印证等）。
- 重排时倾向**更可信来源**，减少「记错还排很前」对信任的伤害；**不以加条数为目标**。

### 2.2 离线慢整理

- **定时批处理**对动态层做合并、拆分、标候选等，与实时对话**解耦**，不占主路径延迟。

### 2.3 类比式召回槽

- 对**弱指代、类比、情绪模式重复**类输入，可走**单独召回策略**，与纯事实检索路径区分。

## 三、情感变化体察（非「禁令条例」式规则）

### 3.1 取向

- 目标不是再叠一套「什么词出现就怎样」的**显性规则**（容易变枷锁），而是让系统多带一点**对「人这一会儿怎么在变」的体感**。
- 真生气却被当成开玩笑、与「玩笑式拒绝被当成永久禁令」**同属一类问题**：都是对**情感阶段与变化方向**读不准。

### 3.2 机制方向（示意，实现可多种形态）

以下均为**增强「变化感」的载体**，而非在用户侧展示一长串条例：

1. **短程情感轨迹**  
   用**极少 token** 维护最近若干轮的整体走向（例如从松到紧、从玩到认真、从堵到软），每轮可更新。模型看到的是**连续曲线的一小段摘要**，不是关键词触发器。

2. **「与上一轮相比」的对照**  
   显式带一句**变化方向**（语气、密度、情绪词层面的对比），帮助区分「还在闹」和「已经当真」——由模型或轻量环节生成**描述性**信息，而不是 if-else 规则表。

3. **生成前的「当下更需要什么」侧写（软）**  
   一行**倾向性提示**（更需承接 / 更需空间 / 可轻松接梗），允许偶错；**不**与永久禁令绑定，**不**把单次判断写成不可撤销的硬状态。

4. **从互动里慢慢长的敏感带（软、慢）**  
   不是「禁词表」，而是：某些互动模式后曾出现情绪陡变，以**低权重、可覆盖**的方式进入侧写或排序微调，随后续正向互动**自然淡化**。

### 3.3 与记忆分层的关系

- 中期事实与约定仍在**动态层**；**情感轨迹 / 对照侧写**可以是**独立短字段、短块或与注入拼接**，核心是**服务当轮体感**，不必事事落成长期条。
- 与已落地的「心事」「检索前方向规划」等可衔接：规划侧重**找什么记忆**，本节侧重**这一会儿人可能处在什么相位**——二者互补，仍避免叠成厚规则书。
