# Notion 建表傻瓜步骤

## 一、一个数据库多分类（书房/客厅/卧室/图书馆）

**说明**：只建**一个** Database，用「分类」这一列区分书房/客厅等，再为每个分类建一个「视图」，看起来就像多个表。

### 步骤

1. **新建 Database**
   - 在 Notion 里新建一页，输入 `/table` 或 `/database`，选 **Table - Full page**（整页表格）。
   - 给这页起个名，例如「归档库」。

2. **改/加属性列（字段）——网关要写，列名必须一致**
   - 第一列默认是 **Title**：把列名改成 **`id`**（网关写的是每条归档的唯一 id）。
   - 点表格右上角 **+** 添加属性，按下面加齐：

   | 列名（必须一致） | 类型 | 说明 |
   |------------------|------|------|
   | **id** | Title | 唯一标识（记忆 id 或轮次标识），网关写入 |
   | **content** | 富文本 / Rich text | 记忆正文或原文，网关写入 |
   | **promoted_at** | 日期 / Date | 可选「包含时间」。含义见下。 |
   | **分类** | 单选 / Select | 选项：`书房`、`客厅`、`卧室`、`图书馆`，网关按 tag 写入 |

   **promoted_at 是什么时间**：这条归档对应的**记忆/轮次的逻辑时间**，不是「网关处理完成的时间」。若归档脚本或 DS 在决策里返回了 `timestamp` 或 `last_mentioned`，网关就用那个时间写入；否则用当前时间。所以批处理时若脚本给每条记忆带上自己的时间，就会按那个时间落库。

   **不需要 importance、mention_count**：网关归档写入时不写这两列，表里可以不建。若建了也会留空。

3. **给「分类」加选项**
   - 点「分类」列头 → Edit property → Select 里逐个加：`书房`、`客厅`、`卧室`、`图书馆`。

4. **每个分类一个视图（看起来像多张表）**
   - 点表格左上角 **+ New view**，选 Table，名字填 **书房**。
   - 在新视图里：点 **Filter** → 添加条件 → 属性选「分类」→ 等于 → 选 **书房**。
   - 再 **+ New view**，名字 **客厅**，Filter 选「分类」= **客厅**。
   - 同理建 **卧室**、**图书馆** 视图。
   - 以后写条目时选好「分类」，在对应视图里就只看到该分类。

5. **拿 Database ID 给网关用**
   - 打开该 Database 页面，浏览器地址栏里有一串 32 位十六进制（有时带 `-`），例如：  
     `https://www.notion.so/workspace/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=...`  
     其中 `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` 就是 Database ID。
   - 环境变量只配**一个**：`NOTION_ARCHIVE_DATABASE_ID=上面这个 ID`。网关会按「分类」写入书房/客厅/卧室/图书馆。

## 二、小本本（按时间排序，最新在上）

**说明**：小本本用**一个 Database**，每条一条记录，有「内容」和「时间」，在 Notion 里按时间降序 = 最新在最上面。

### 步骤

1. **新建 Database**
   - `/table` → **Table - Full page**，起名「小本本」。

2. **属性列（字段）只保留/建这两个**

   | 列名（必须一致） | 类型 | 说明 |
   |------------------|------|------|
   | **内容** | Title | 记的那句话/那段字。若默认第一列叫 "Name"，点列头 Rename 改成 **内容** |
   | **时间** | Date | 选 **Include time**（包含时间），用于排序 |

   - 其他列可以删掉或隐藏（例如不需要的 "Tags"）：点列头 → **Delete property** 或 **Hide in view**。

3. **时间列设置**
   - 点「时间」列头 → **Edit property** → Date → 打开 **Include time**，保存。

4. **视图排序（最新在上）**
   - 点表格左上角视图名旁 **⋯** 或 **Sort** → **Add sort** → 选 **时间** → 选 **Descending**（降序）。  
   - 保存后，新记的（时间最新）会排在最上面。

5. **拿 Database ID**
   - 同上面，从该小本本页面的 URL 里复制 32 位十六进制 ID，配到环境变量 **`NOTION_NOTEBOOK_DATABASE_ID`**。

## 三、字段属性速查

### 记忆库（一个数据库多分类，网关归档用）

| 属性名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Title | 是 | 唯一标识，网关写 |
| content | 富文本 | 是 | 记忆正文，网关写 |
| promoted_at | 日期 | 建议 | 记忆/轮次的逻辑时间（见上），网关写 |
| 分类 | Select | 是 | 书房、客厅、卧室、图书馆，网关写 |

网关**不写** importance、mention_count，表里可不建这两列。

### 小本本

| 属性名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 内容 | Title | 是 | 记的那条内容（网关写这里） |
| 时间 | Date（Include time） | 是 | 用于排序，网关写渡传的 note_time 或当前时间 |

---

## 四、网关环境变量对应关系

- **小本本 Database**：`NOTION_NOTEBOOK_DATABASE_ID` = 小本本那一页的 Database ID；列名必须为 **内容**、**时间**。
- **归档（一个库多分类）**：只配 `NOTION_ARCHIVE_DATABASE_ID`，表里要有 **id、content、importance、mention_count、promoted_at、分类**。不配则用原来的四张表（四个 ID）。

文档里写的列名必须和 Notion 里完全一致，网关是按名字读写的。

## 五、记忆库 和 核心记忆 在 Notion 里关联（关系列）

**说明**：在 Notion 里把「记忆库」和「核心记忆」用**关系**连起来，这样在核心记忆里点一条，能看到它关联了记忆库里的哪些条目；反过来在记忆库里也能看到连到哪条核心记忆。

### 步骤（傻瓜版）

1. **打开「核心记忆」这个数据库**
   - 在侧边栏点进核心记忆那一页（就是 `NOTION_CORE_CACHE_DATABASE_ID` 对应的那个表）。

2. **加一列「关系」**
   - 点表头最右边 **+**（添加属性）。
   - 属性类型选 **「关系」**（Relation）。
   - 关系选 **「关联到：记忆库」**（选你建的那个记忆库/归档库的 Database）。
   - 列名随便起，例如 **「关联记忆」** 或 **「记忆库」**，保存。

3. **要不要「双向显示」**
   - 加完关系后 Notion 会问：要不要在「记忆库」里也显示这个关系？
   - 选 **「在记忆库中显示」**，并给记忆库那边显示的列起个名，例如 **「来自核心记忆」**。这样记忆库表里会多一列「来自核心记忆」，可以看见这条记忆被哪条核心记忆关联了。

4. **以后怎么用**
   - 在核心记忆里：每一行可以在「关联记忆」列里点 **+**，选记忆库里的某一条，就关联上了。
   - 在记忆库里：若你选了「在记忆库中显示」，会多一列「来自核心记忆」，能看到这条记忆被哪些核心记忆标记。

5. **注意**
   - 关系是手动在 Notion 里点的（或以后用 API 写）；网关归档只会往记忆库写新行，不会自动给核心记忆填关系。你要在 Notion 里自己把「核心记忆的某条」和「记忆库的某条」连起来。
   - 若你想「核心记忆里每条对应记忆库里的哪条」由网关或脚本自动填，需要另外做：网关在写记忆库时拿到新页的 id，再在核心记忆里更新对应行的关系。当前步骤只教在 Notion 里手动关联。
