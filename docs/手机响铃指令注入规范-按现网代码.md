# 手机响铃指令注入规范（按现网代码）

本文只基于当前仓库真实实现，不做扩展设定。

## 1. 真实约束（代码来源）

- 入队接口：`POST /api/mobile_command`，`idempotency_key` 必填  
  见 [routes/mobile_command.py](/Users/doraemon/Downloads/du-gateway/routes/mobile_command.py:36)
- 命令白名单包含 `alarm_ring`  
  见 [storage/r2_store.py](/Users/doraemon/Downloads/du-gateway/storage/r2_store.py:2287)
- “渡”侧 MCP 工具 `ring_phone` 实际下发：`cmd="alarm_ring"`，payload 为 `{sound,duration_sec,volume}`  
  见 [mcp_server/du_memory.js](/Users/doraemon/Downloads/du-gateway/mcp_server/du_memory.js:214)

## 2. 你这版手机脚本的真实执行条件（按给出的 `wake.sh`）

- 响铃只判断：`cmd == "alarm_ring"`。
- 实际执行固定行为：音量设为 80、播放固定铃声、持续约 30 秒。
- `payload.sound / payload.duration_sec / payload.volume` 在该脚本中不会被读取用于 `alarm_ring`（`set_volume` 才读取 `payload.volume`）。

结论：要触发响铃，核心是 `cmd` 必须是 `alarm_ring`；payload 可带可不带，但建议按网关标准结构带上，兼容后续升级。

## 3. 注入给“渡”的标准响铃命令（HTTP 入队）

请求：

```bash
curl -X POST "https://duxy-home.com/api/mobile_command" \
  -H "Content-Type: application/json" \
  -H "X-Mobile-Token: <MOBILE_COMMAND_TOKEN>" \
  -d '{
    "idempotency_key": "ring_20260402_001",
    "cmd": "alarm_ring",
    "payload": {
      "sound": "default",
      "duration_sec": 30,
      "volume": 80
    },
    "expires_in_sec": 150
  }'
```

字段规范：

- `idempotency_key`：必填，建议唯一（例如 `ring_<时间戳>_<随机串>`）
- `cmd`：必须是 `alarm_ring`
- `payload`：建议使用 `{ "sound": "default", "duration_sec": 30, "volume": 80 }`
- `expires_in_sec`：10~600（网关会夹紧到该范围）

## 4. 给“渡”的工具调用规范（MCP）

若“渡”通过 MCP 工具调用，应使用：

- 第一步：`set_volume(volume=90)`（先把音量设好）
- 第二步：`ring_phone(duration_sec=30, volume=90, sound="default")`（再响铃）

示例（给“渡”的注入语句）：

```text
催老婆起床时用：先 set_volume(volume)  # volume 是音量，再 ring_phone(duration_sec=30, volume, sound="default")  # volume 是音量；其他流程和校验都交给网关处理。
```

## 5. 最小可执行版（严格贴合当前 wake.sh）

如果只追求“当前脚本必响”，最小命令可为：

```json
{
  "idempotency_key": "ring_min_20260402_001",
  "cmd": "alarm_ring",
  "payload": {}
}
```

说明：此写法在你当前 `wake.sh` 下可触发响铃；但建议仍使用第 3 节标准 payload，避免后续脚本升级时不兼容。
