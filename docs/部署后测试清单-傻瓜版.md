# 部署后测试清单（傻瓜版）

部署完成后，按下面顺序测一遍即可。

---

## 一、自动测（一条命令）

在服务器上进入项目目录，执行：

```bash
python scripts/call_test.py
```

**看什么**：6 步都打印「通过」或状态 200 即算通过。

| 步骤 | 在测什么 |
|------|----------|
| 1 | 网关活着（/health） |
| 2 | 能拉到模型列表（/v1/models） |
| 3 | 能正常聊天转发（/v1/chat/completions） |
| 4 | 管理端「最近窗口」能打开（/admin/windows） |
| 5 | 白名单接口正常（/admin/whitelist） |
| 6 | 黑名单接口正常（/admin/blacklist） |

---

## 二、人工测（按需勾选）

### 1. RikkaHub 接网关

- **测什么**：用 RikkaHub 真实对话是否走网关、是否正常。
- **怎么测**：  
  1. RikkaHub 里 API 地址填你的网关地址（如 `https://你的域名/v1`）。  
  2. API Key 随便填（网关不校验）。  
  3. 选一个模型，发一条消息。  
  4. 能收到正常回复 → 通过。

---

### 2. 黑名单：回复带「（黑名单）」  

- **测什么**：黑名单窗口的回复末尾是否带「（黑名单）」。
- **怎么测**：  
  1. 管理端或接口把某个 window_id 加入黑名单。  
  2. 用这个 window_id 发一条聊天（Header 里 `X-Window-Id` 填该 id）。  
  3. 看回复正文最后是否有「（黑名单）」→ 有即通过。

---

### 3. 新窗口发「测试」会进黑名单  

- **测什么**：新窗口且第一条消息含「测试」时，是否自动进黑名单。
- **怎么测**：  
  1. 用一个从未用过的 window_id（新窗），Header `X-Window-Id` 填这个 id。  
  2. 第一条消息内容写「测试」或包含「测试」。  
  3. 再发一条消息，看回复是否只有转发内容且末尾有「（黑名单）」→ 是即通过。

---

### 4. 多目标 fallback（配了多个 TARGET 时）

- **测什么**：第一个上游挂了或非 2xx 时，是否自动用下一个。
- **怎么测**：  
  1. 配置里填两个 `TARGET_AI_URLS`（第一个故意填错或关掉）。  
  2. 发一条聊天。  
  3. 能正常拿到 2xx 回复 → 通过。

---

### 5. 时间注入（今天、周几、间隔提示）

- **测什么**：模型是否知道「今天周几」、以及「x 分钟后老婆终于回我了」这类提示。
- **怎么测**：  
  同一窗口先发一条，等超过 30 分钟再发一条，看回复里是否像知道「过了很久才回」；或看日志/注入内容里是否有「周几」「分钟后」等 → 有即通过。

---

### 6. 每 4 轮总结（配了 DEEPSEEK 时）

- **测什么**：白名单窗口聊满 4 轮后是否写了总结。
- **怎么测**：  
  同一 window_id、白名单内，连续对话 4 轮（2 轮 user+assistant 算 2 轮）。看 R2 里该窗口的 summary 或全局总结是否更新 → 更新即通过。

---

### 7. 小本本 / 卧室 / Notion 同步

- **测什么**：小本本更新、卧室通道、核心缓存同步到 Notion 是否生效。
- **怎么测**：  
  - 小本本：对话里触发一次「📝小本本更新」相关，看 R2 的 notebook 或 Notion 是否有新内容。  
  - 卧室：触发一次带「卧室」的内容，看 Notion 卧室页是否有追加。  
  - 同步：调管理端「同步到 Notion / 从 Notion 拉取」，看 Notion 数据库是否和网关一致。

---

## 三、快速对照表

| 项目 | 怎么测 | 通过标准 |
|------|--------|----------|
| 自动测 | 运行 `python scripts/call_test.py` | 6 步都 200/通过 |
| RikkaHub | 填网关地址发一条 | 能收到回复 |
| 黑名单后缀 | 黑名单 window 发一条 | 回复末尾有「（黑名单）」 |
| 新窗+「测试」 | 新 window_id，首条「测试」再发一条 | 回复带「（黑名单）」 |
| 多目标 | 第一个 TARGET 不可用 | 仍能 2xx 回复 |
| 时间/间隔 | 同窗 30 分钟后再发 | 有周几/间隔感 |
| 4 轮总结 | 白名单同窗 4 轮 | R2 summary 更新 |
| 小本本/卧室/Notion | 触发对应功能 | 对应存储有更新 |

---

**总结**：先跑脚本，再按需勾选上面「人工测」；每项按「怎么测」做一遍，满足「通过标准」就打勾。
