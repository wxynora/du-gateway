# 怎么知道各功能有没有正常运行（傻瓜版）

用一个接口就能看到：总结、R2、动态层、核心缓存、小本本、白名单/黑名单、最近窗口、上次回复时间、各项配置是否填了。

---

## 一步：打开状态页

在浏览器地址栏输入（**把「你的网关地址」换成真实地址**）：

```
你的网关地址/admin/status
```

例如本机：`http://localhost:5000/admin/status`  
例如云上：`https://你的网关域名/admin/status`

或用命令行：

```powershell
curl "http://localhost:5000/admin/status"
```

会返回一坨 JSON，下面说明每个字段表示什么。

---

## 返回里各字段含义

| 字段 | 表示什么 | 正常/有数据时 |
|------|----------|----------------|
| **summary** | 总结记忆（每 4 轮 DeepSeek 更新） | `ok: true`，`has_summary: true` 且 `length > 0` 表示已有总结 |
| **r2** | 云存储 R2 是否能用 | `ok: true`，`message: "可读"` |
| **dynamic_memory** | 动态层记忆条数 | `ok: true`，`count` 为条数（0 也正常，表示还没记） |
| **core_cache** | 核心缓存待审条数 | `ok: true`，`pending_count` 为条数 |
| **notebook** | 小本本条数 | `ok: true`，`count` 为条数 |
| **whitelist** | 白名单窗口数 | `ok: true`，`count` 为个数 |
| **blacklist** | 黑名单窗口数 | `ok: true`，`count` 为个数 |
| **recent_windows** | 最近出现过的窗口数 | `ok: true`，`count` 为个数 |
| **last_user_reply** | 是否记录过「上次 user 回复时间」 | `ok: true`，`exists: true` 表示已有记录 |
| **config** | 配置是否填了（不测是否真的能用） | `target_configured`、`summary_deepseek_configured`、`notion_configured`、`embedding_configured` 为 `true` 表示对应项已配置 |

某个功能异常时，对应块里会有 `ok: false` 和 `error: "..."`，方便排查。

---

## 示例（正常时大概长这样）

```json
{
  "summary": { "ok": true, "has_summary": true, "length": 320 },
  "r2": { "ok": true, "message": "可读" },
  "dynamic_memory": { "ok": true, "count": 5 },
  "core_cache": { "ok": true, "pending_count": 2 },
  "notebook": { "ok": true, "count": 0 },
  "whitelist": { "ok": true, "count": 1 },
  "blacklist": { "ok": true, "count": 0 },
  "recent_windows": { "ok": true, "count": 3 },
  "last_user_reply": { "ok": true, "exists": true },
  "config": {
    "target_configured": true,
    "summary_deepseek_configured": true,
    "notion_configured": false,
    "embedding_configured": true
  }
}
```

看这一屏就能知道各功能有没有在跑、有没有数据。
