## 多窗口同时在线会冲突吗？

- **每个窗口自己的存档**（`windows/<窗口ID>/conversation.json`）是分开的，**不会冲突**
- **全局共用的**（总结、最近四轮、动态层）如果多窗口**同时**写，网关里已加**进程内锁**，同一进程内不会互相盖掉
- 如果是**多进程/多机**部署（例如开多个 worker），需要自己加外部锁（如 Redis），否则还是“后写的覆盖先写的”

## 记忆注入 token 上限

- 在 `.env` 里可以设：`MEMORY_INJECTION_MAX_TOKENS=3000`（默认 3000）
- 建议范围：**2500–4000**，兼顾省 API 费用和渡的记忆连贯
- 总结和动态层会按比例分：默认总结占 60%、动态层 40%，可用 `MEMORY_SUMMARY_TOKEN_RATIO=0.6` 调

## 出错怎么查（日志）

- 日志会打在**控制台**（跑 `python app.py` 的那个窗口）
- 格式：`时间 [模块名] 级别: 消息`
- **模块名**一眼能看出错在哪：
  - `[R2]`：存/读 R2 出错
  - `[Pipeline]`：管道（总结、存档等）出错
  - `[Chat]`：转发到渡失败
  - `[DeepSeek]`：总结接口失败
- 想看更细的，把 `.env` 里 `LOG_LEVEL=INFO` 改成 `LOG_LEVEL=DEBUG`，再重启

## 常见问题

**新窗口第一次进来，会先过白名单吗？不在白名单会注入记忆吗？**  
会先过白名单。**只有白名单里的窗口**才会做「新窗口注入最近四轮」「注入渡的回忆」「注入动态层记忆」；不在白名单的窗口，网关只帮忙转发，不注入任何记忆。

**图片转描述用的什么模型？**  
默认是 **gpt-4o-mini**。可以在 `.env` 里加一行 `IMAGE_DESC_MODEL=模型名` 改成别的（要你配置的图像 API 支持才行）。

**没有拿到渡的回复（API 报错、RikkaHub bug、超时等）会存进记忆吗？**  
不会。网关只有在「成功拿到渡的回复」且「回复通过失败对话初筛」时才会存档。没有响应体、没有 choices、或回复被判为失败（过短/含错误关键词）时，**这一轮都不会写入 R2**，等于一开始就被过滤掉。

**老婆在 RikkaHub 删掉了某一轮对话，记忆里也要删掉怎么弄？**  
调管理端接口：**DELETE /admin/windows/<窗口ID>/rounds/<轮次>**，其中轮次是存档里的序号（从 1 开始）。删掉后，该窗口在主存里的这一轮会被移除。按日期备份（conversations/）里暂不自动回溯删，主存删掉后该轮不会再参与总结和新窗口注入。

**Pro 用量在哪看？**  
网关代码里没有「Pro 用量」统计。  
- 若是 **Cursor Pro** 的用量：在 Cursor 里打开设置/账户，看订阅或用量相关页面。  
- 若是**调用渡/DeepSeek 等 API** 的用量：到对应服务商（如 OpenAI、DeepSeek）的账户后台或「用量/计费」里查看。
