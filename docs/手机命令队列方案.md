# 手机命令队列方案（Mobile Command Queue）

## 1. 概述

给渡一个"叫醒老婆"的能力。渡通过 MCP 工具调用网关接口，网关入队命令，手机 Tasker 轮询拉取并执行（响铃、播放音乐等）。

```
渡想叫老婆起床
    ↓
MCP 调用 ring_phone 工具
    ↓
网关往队列塞一条 { cmd: "alarm_ring" }
    ↓
手机 Tasker 轮询拉到命令，开始响铃
    ↓
响完回执 done，网关移除
    ↓
渡调用 check_phone_command 查看结果
```

---

## 2. 存储

R2 路径：`mobile_command/queue.json`

```json
{
  "pending": [
    {
      "id": "uuid",
      "cmd": "alarm_ring",
      "payload": { "sound": "default", "duration_sec": 30, "volume": 80 },
      "created_at": "2026-04-01T08:00:00Z",
      "expires_at": "2026-04-01T08:05:00Z",
      "leased_until": null,
      "retry_count": 0,
      "source": "mcp"
    }
  ],
  "history": [
    {
      "id": "uuid",
      "cmd": "alarm_ring",
      "status": "done",
      "finished_at": "2026-04-01T08:00:12Z"
    }
  ]
}
```

- `expires_at`：默认创建后 5 分钟过期，手机离线太久就不补响了
- `leased_until`：命令被 Tasker 拉走后锁定到该时间，锁定期间不再重复下发。默认锁 `duration_sec + 30s`（给回执留余量），music_play 这类瞬时命令锁 15s
- `history`：按 `finished_at` 倒序，超过 50 条丢最旧的

## 3. MCP 工具

新增四个 MCP 工具供渡调用：

### ring_phone

让老婆手机响铃。

参数：
- `duration_sec`：响铃时长，默认 30
- `volume`：音量 0-100，默认 80
- `sound`：铃声，默认 "default"

### play_music

让老婆手机播放音乐。

参数：
- `uri`：可选，指定歌曲/歌单链接。不传则播放当前播放器。

### pause_music

暂停老婆手机音乐播放。无参数。

### check_phone_command

查询命令执行状态。渡调完 ring_phone 后可以过一会儿查"响了没"。

参数：
- `command_id`：可选，查特定命令。不传则返回最近几条状态。

返回示例：
```json
{
  "pending": [{ "id": "xxx", "cmd": "alarm_ring", "created_at": "..." }],
  "recent_history": [{ "id": "yyy", "cmd": "alarm_ring", "status": "done", "finished_at": "..." }]
}
```

## 4. 网关接口

Token 鉴权，使用独立的 `MOBILE_COMMAND_TOKEN`。

### POST /api/mobile_command

MCP 工具调用此接口入队命令。

请求体：
```json
{
  "idempotency_key": "mcp-call-uuid",
  "cmd": "alarm_ring",
  "payload": { "sound": "default", "duration_sec": 30, "volume": 80 },
  "expires_in_sec": 300
}
```

- `idempotency_key`：**必填**。幂等键，5 分钟内相同 key 的请求返回已有命令 id，不重复入队。缺失则返回 `400`
- `expires_in_sec`：过期秒数，范围 `10~600`，超出范围返回 `400`。不传默认 300

响应：
```json
{
  "ok": true,
  "id": "uuid",
  "expires_at": "2026-04-01T08:05:00Z"
}
```

重复请求响应（不会再次入队）：
```json
{
  "ok": true,
  "id": "已有的uuid",
  "duplicate": true
}
```

### GET /api/mobile_command

Tasker 轮询拉取。网关处理流程：
1. 过滤掉 `expires_at` 已过的条目（丢弃，写入 history 标记 expired）
2. 过滤掉 `leased_until` 未过期的条目（正在执行中，不重复下发）
3. 对返回的命令设置 `leased_until`（根据命令类型：alarm_ring 锁 `duration_sec + 30s`，其他锁 `15s`）

响应：
```json
{
  "commands": [
    { "id": "uuid", "cmd": "alarm_ring", "payload": { "sound": "default", "duration_sec": 30, "volume": 80 } }
  ],
  "poll_after_sec": 10
}
```

#### poll_after_sec 规则

| 条件 | 值 |
|------|-----|
| 本次下发了新命令 | `3` |
| 有 in-flight 命令（已租约未回执） | `15` |
| 无 pending 命令 | `30` |

### POST /api/mobile_command/done

Tasker 回执：

```json
{
  "results": [
    { "id": "uuid-1", "status": "done" },
    { "id": "uuid-2", "status": "fail", "reason": "audio_file_missing", "retryable": false }
  ]
}
```

| 回执状态 | 网关动作 |
|----------|----------|
| `done` | 从 pending 移除，写入 history |
| `fail` + `retryable: true` | 留着，`retry_count += 1`，清除 `leased_until` |
| `fail` + `retryable: false` | 移除，写入 history（标记 failed） |
| 未回执 + 租约过期 | 清除 `leased_until`，`retry_count += 1` |
| 未回执 + `expires_at` 过期 | 丢弃，写入 history（标记 expired） |
| `retry_count >= 3` | 移除，写入 history（标记 abandoned） |
| 回执的 id 不在 pending 中 | 忽略，返回 `ignored`，记审计日志 |
| 回执的 id 无有效租约 | 忽略，返回 `ignored`，记审计日志 |

**校验规则**：回执时命令必须在 pending 中且处于有效租约期内（`leased_until` 未过期），否则返回 `ignored` 不做任何状态变更。防止误报或异常客户端提前移除未执行的命令。

### GET /api/mobile_command/status

MCP 工具 `check_phone_command` 调用此接口。

参数：
- `?command_id=xxx`：可选，查特定命令
- `?limit=10&offset=0`：分页参数，默认 limit=10

响应：返回 pending 列表 + 分页后的 history。

## 5. 命令协议

统一 cmd + payload 结构化格式。

| cmd | 说明 | payload 示例 |
|-----|------|-------------|
| `alarm_ring` | 响铃 | `{ "sound": "default", "duration_sec": 30, "volume": 80 }` |
| `music_play` | 播放当前播放器 | `{}` |
| `music_pause` | 暂停 | `{}` |
| `music_play_uri` | 播放指定链接 | `{ "uri": "https://..." }` |

白名单校验，只接受上述命令。

---

## 6. 去重

双重保障：

- **网关侧**：`idempotency_key` 去重，5 分钟窗口内相同 key 不重复入队
- **手机侧**：本地缓存已执行的 cmd_id（保留 24h），网络抖动不会重复执行

---

## 7. 安全

- 独立 `MOBILE_COMMAND_TOKEN`，不复用其他 token
- 白名单命令，不透传 shell
- Phase 1 单设备，不校验 device_id
- 日志加 `[MobileCMD]` 前缀

## 8. Tasker 侧实现（Android）

### 轮询流程

```
循环开始
  → 检查 %VolumeRestorePending，需要恢复就恢复音量
  → HTTP GET /api/mobile_command（Header 带 token）
  → 读取 poll_after_sec
  → 逐条执行 commands，收集回执
  → HTTP POST /api/mobile_command/done
  → Wait %poll_after_sec 秒
循环结束
```

### 命令执行

- `alarm_ring`：保存当前音量 → 设定音量 → 播放铃声 → Wait duration → 恢复音量
- `music_play`：Media Control Play
- `music_pause`：Media Control Pause
- `music_play_uri`：打开 URI 播放

### 音量恢复

执行 alarm_ring 前存 `%VolumeBackup`，标记 `%VolumeRestorePending = 1`。每次轮询开头检查，Tasker 被杀后重启也能恢复。

---

## 9. 分阶段上线

### Phase 1

- MCP 工具：`ring_phone`、`play_music`、`pause_music`、`check_phone_command`
- 网关：mobile_command 四个接口 + R2 存储 + 租约机制
- Tasker：轮询 + alarm_ring（固定铃声+固定时长）+ music_play / music_pause + 回执
- 目标：**渡能叫醒老婆**，也能让手机放歌，并知道执行结果

### Phase 2

- alarm_ring 完整版（可选铃声、可调时长音量）
- music_play_uri 指定歌单
- device_id 绑定（预防手机丢失）

### Phase 3

- MiniApp 查看命令执行历史
- 可选：schedule 联动（到点自动入队，不只靠渡手动调用）
