## 二、可能出 bug

| 项 | 说明 |
|----|------|
| **问句判断偏松** | 兜底时间看「渡的上一轮是问句 + 含几点/时间/现在」；问句用「吗/呢/啊/呀」判断，语气词多时可能误触，导致具体时间注入偏多。 |
| **动态层 JSON 解析失败** | DS 返回被 markdown 包住或多字少字时，解析失败会整轮当 skip，该记的没记；已有重试 1 次 + warning 日志。 |
| **卧室完全靠 DS 的 tag** | DS 没给 `tag: "卧室"` 就不会走卧室通道，卧室内容会进动态层；可考虑加关键词兜底（如原文含「卧室」）。 |
| **R2/Notion 写失败但接口 200** | 小本本、卧室 Notion 已加重试 + error 日志；其它写（存档、总结、动态层）失败时请求已 200，前端不知道，可考虑关键写失败告警。 |
| **总结被 token 截断** | 注入受 `MEMORY_INJECTION_MAX_TOKENS` 限制，总结过长会从尾部截掉，可能把最近内容截没；prompt 已要求总篇幅约 2200 字内。 |
| **Notion 同步列名/类型不一致** | 核心缓存 Notion 列名必须与代码约定一致（id/content/promoted_by/importance/mention_count/promoted_at/tag），类型要对（Title/Text/Select/Number/Date），否则同步报错或丢字段。 |

## 三、已落地的（供对照）

- 核心缓存：自动提拔、GET/DELETE、Notion 剪切粘贴同步、tag 字段。
- 动态层：终稿全文 + 兜底规则、卧室定义（亲密互动/限制性语言）、单条应用、merge 兜底、JSON 清洗 + 重试。
- 时间：全部北京时间；常驻今日+大概时段；兜底时间/农历按渡上一轮触发；get_time_info 提示。
- 小本本：📝+「小本本更新」才截取；只对纯文本判断渡上一轮；R2/Notion 写失败 error+重试。
- 失败回复过滤：过短+含错误关键词才过滤。
