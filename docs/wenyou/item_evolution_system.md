# 文游物品、装备与进化系统

本文档从 `docs/wenyou_rules.md` 拆出，专门维护道具、商店、抽卡、武器装备、锻造回收，以及原“血统”改名后的进化系统。

核心原则：

- 目录字段保持简单：名字、等级、分类、作用效果、价格。
- 名字可以荒诞、生活化、怪谈化，但作用效果必须能被系统判定。
- GM 只描述道具使用后的画面，不计算数值。
- 后端规则引擎负责扣次数、扣耐久、扣代价、加状态、改结算。
- 武器、防具、饰品、可装备工具统一有耐久。
- 原“血统”不再作为血统系统，改为“进化系统”：玩家身体强化到阶段时，可以选择继续保持人类方向，或向其他生物/异常方向进化。

## 道具目录

最终道具目录只要求五个本体字段：

| 字段 | 说明 |
| --- | --- |
| `name` | 道具名字 |
| `rarity` | 等级/稀有度：D/C/B/A/S |
| `category` | 用途分类，不写武器/防具/装备这种形态 |
| `effect` | 作用效果，必须能被系统规则结算；武器、防具、饰品、可装备工具要在效果里写明形态、槽位和耐久 |
| `price` | 商店基础价格；不可购买道具也用于回收和奖励估值；不由 DS 生成，完整列表审校后统一补 |

## SQL 表设计

道具目录应该落成 SQL 表或等价的可查询内容库，而不是运行时从 Markdown 全量读取。Markdown 只做设计文档；后端实际抽卡、商店轮转、奖励掉落都查内容表。

核心拆分：

- `item_catalog`：道具定义表，存所有可抽取、可售卖、可掉落的道具模板。
- `player_item_instances`：玩家背包实例表，存玩家实际拥有的某个道具、剩余次数、耐久、封印状态等。
- `shop_rotations`：商店轮转表，存某天/某次结算刷出来的货架，不每次打开商店都重新抽。
- `gacha_pool_items`：可选表，用于覆盖某个卡池的权重、限定标签和上架状态。

### item_catalog

道具定义表。商店、抽卡、奖励都从这里按条件取。

```sql
CREATE TABLE item_catalog (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  rarity TEXT NOT NULL CHECK (rarity IN ('D', 'C', 'B', 'A', 'S')),
  item_type TEXT NOT NULL DEFAULT 'consumable'
    CHECK (item_type IN ('consumable', 'weapon', 'armor', 'accessory', 'equippable_tool', 'material', 'special')),
  equip_slot TEXT
    CHECK (equip_slot IS NULL OR equip_slot IN ('main_weapon', 'offhand_weapon', 'armor', 'accessory', 'special')),
  category TEXT NOT NULL,
  effect_json TEXT NOT NULL,
  requirements_json TEXT NOT NULL DEFAULT '{}',
  use_cost_json TEXT NOT NULL DEFAULT '{}',
  tags_json TEXT NOT NULL DEFAULT '[]',
  use_phase_json TEXT NOT NULL DEFAULT '[]',
  consume INTEGER NOT NULL DEFAULT 1,
  stackable INTEGER NOT NULL DEFAULT 0,
  shop_allowed INTEGER NOT NULL DEFAULT 0,
  gacha_allowed INTEGER NOT NULL DEFAULT 0,
  seal_rank TEXT,
  price INTEGER NOT NULL DEFAULT 0,
  weight INTEGER NOT NULL DEFAULT 100,
  enabled INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

推荐索引：

```sql
CREATE INDEX idx_item_catalog_shop ON item_catalog (shop_allowed, enabled, rarity);
CREATE INDEX idx_item_catalog_gacha ON item_catalog (gacha_allowed, enabled, rarity);
CREATE INDEX idx_item_catalog_category ON item_catalog (category, rarity);
CREATE INDEX idx_item_catalog_type ON item_catalog (item_type, rarity);
```

说明：

- `effect_json` 存系统可执行效果。
- `category` 是用途分类，例如增益、防护、探查、线索；不用于表示武器或防具。
- `item_type/equip_slot` 表示物品形态和装备槽位，例如马桶搋子是 `weapon/main_weapon`，不锈钢盆是 `armor/armor`。
- `requirements_json` 存使用/装备限制，例如力量、体力、敏捷、智力、精神力基础值、当前精神力、当前 SAN、阶位、能力或进化方向。
- `use_cost_json` 存每次触发的固定代价，例如 SAN 损失、污染增加、威胁时钟推进、耐久额外消耗。
- `tags_json` 存副本主题、活动、怪谈类型等标签。
- `weight` 用于同稀有度内加权抽取，不影响先前定义的 D/C/B/A/S 概率。
- `price` 是道具目录基础价格，用于商店、回收、奖励估值。
- A/S 道具可以在表里存在，但普通商店查询必须过滤掉。

### player_item_instances

玩家背包实例表。目录表只定义“这是什么”，实例表记录“玩家手里这一件还剩多少”。

```sql
CREATE TABLE player_item_instances (
  uid TEXT PRIMARY KEY,
  player_id TEXT NOT NULL,
  item_id TEXT NOT NULL REFERENCES item_catalog(id),
  quantity INTEGER NOT NULL DEFAULT 1,
  uses_left INTEGER,
  durability INTEGER,
  durability_max INTEGER,
  sealed INTEGER NOT NULL DEFAULT 0,
  broken INTEGER NOT NULL DEFAULT 0,
  source TEXT,
  instance_state_json TEXT NOT NULL DEFAULT '{}',
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

规则：

- 可堆叠一次性道具可以用 `quantity`。
- 多次使用道具用 `uses_left`。
- 武器、防具、饰品、可装备工具用 `durability/durability_max`。
- 高阶封印道具用 `sealed = 1`，使用时走低配或拒绝。
- `instance_state_json` 存特殊标记，例如未知标记、绑定副本、临时诅咒。

### shop_rotations

商店轮转表。商店刷新时生成一次，打开商店时只读已生成货架。

```sql
CREATE TABLE shop_rotations (
  id TEXT PRIMARY KEY,
  player_id TEXT,
  shop_id TEXT NOT NULL,
  phase TEXT NOT NULL,
  items_json TEXT NOT NULL,
  expires_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

`items_json` 可以保存：

```json
[
  { "item_id": "bandage", "price": 25, "stock": 3 },
  { "item_id": "old_key", "price": 90, "stock": 1 }
]
```

### gacha_pool_items

如果不同卡池需要覆盖权重，可以建单独表；简单实现也可以直接用 `item_catalog.gacha_allowed`。

```sql
CREATE TABLE gacha_pool_items (
  pool_id TEXT NOT NULL,
  item_id TEXT NOT NULL REFERENCES item_catalog(id),
  weight INTEGER NOT NULL DEFAULT 100,
  limited INTEGER NOT NULL DEFAULT 0,
  enabled INTEGER NOT NULL DEFAULT 1,
  PRIMARY KEY (pool_id, item_id)
);
```

## 随机抽取策略

不要每次从 Markdown 读全量。

推荐流程：

1. 服务启动或内容包更新时，把 `item_catalog` 加载到内存缓存。
2. 抽卡、奖励、商店先按稀有度/分类/物品形态/标签/开关过滤候选。
3. 候选集通常很小，可以在内存里按 `weight` 加权随机。
4. 如果后端不做缓存，也应该用 SQL 条件查询候选，不读全表文档。

商店候选：

```sql
SELECT *
FROM item_catalog
WHERE enabled = 1
  AND shop_allowed = 1
  AND rarity IN ('D', 'C')
ORDER BY RANDOM()
LIMIT 8;
```

B 级偶尔出现时：

```sql
SELECT *
FROM item_catalog
WHERE enabled = 1
  AND shop_allowed = 1
  AND rarity = 'B'
ORDER BY RANDOM()
LIMIT 1;
```

抽卡候选：

```sql
SELECT *
FROM item_catalog
WHERE enabled = 1
  AND gacha_allowed = 1
  AND rarity = :rolled_rarity;
```

奖励候选：

```sql
SELECT *
FROM item_catalog
WHERE enabled = 1
  AND rarity = :reward_rarity
  AND gacha_allowed = 1;
```

实现建议：

- 小内容包可以全量缓存 `item_catalog`，但缓存来自 SQL 表，不来自 Markdown。
- 内容包较大时按 `rarity/category/item_type/tags` 查询候选，再在候选里加权随机。
- `ORDER BY RANDOM()` 对小表足够；大表应在应用层用权重随机，或用预计算随机 key。
- 所有随机必须记录 seed、候选范围、最终 item_id，方便复盘和 debug。

## 道具分类

`category` 只写一级用途大类，不按具体数值效果拆。HP 回复、SAN 回复、替死、回满、撤离这类具体行为写进 `effect`。

武器、防具、饰品、可装备工具不是 `category`，而是 `item_type/equip_slot`。也就是说：

- `防护类` 是用途：抵消伤害、污染、误认、死亡后果。
- `防具` 是物品形态：穿戴或持有后占用装备槽、有耐久。
- 一件防具通常是 `category = 防护类`，但也可以是 `伪装类`、`探查类` 或 `规则类`，取决于它主要提供什么效果。
- 一件武器不是单独写成“武器类”，而是按主要用途选择 `category`。

武器用途分类：

| 武器主要效果 | 推荐 `category` | 示例 |
| --- | --- | --- |
| 打击、破坏、开路、威慑 | 工具类 | 马桶搋子、消防斧、折叠椅、撬棍 |
| 格挡、反击、保护队友 | 防护类 | 锅盖盾、圣水枪、镇宅木剑 |
| 侦测异常、照明、定位实体 | 探查类 | 探灵手电、骨哨、回声弩 |
| 固定证据、钉住证言、封存记忆 | 线索类 | 录音匕首、证言钉枪 |
| 身份遮蔽、恐吓、伪装成普通物 | 伪装类 | 校工扫帚、保安电棍、工牌刀 |
| 规则干涉、污染、邪神/怪谈器官 | 规则类 | 邪神的心脏、黑门骨刺、逆十字钉 |
| 影响撤离、开门、跨节点移动 | 位移类 | 门缝短刀、车票弩 |

| 分类 | 说明 | 常见作用效果 |
| --- | --- | --- |
| 增益类 | 回复、临时强化、抗性、状态缓解 | 回复 HP/SAN、判定加值、抗性提高、移除轻伤/动摇 |
| 防护类 | 抵消伤害、误认、精神暗示或死亡后果 | 减伤、抵消精神暗示、替死、污染防护 |
| 探查类 | 发现异常、危险、机关、实体靠近 | 危险提示、异常扫描、陷阱侦测、实体靠近提示 |
| 线索类 | 验证、保存、定位或补全线索 | 验证证言、保存记忆、补全规则片段、定位关键物 |
| 工具类 | 开锁、攀爬、破坏、追踪、维修、照明 | 开锁加值、管道/堵塞判定加值、追踪加值、照明 |
| 伪装类 | 身份遮蔽、降低暴露度或改变被识别方式 | 暴露度降低、身份遮蔽、路人化 |
| 位移类 | 移动、脱困、跨越障碍、争取撤离机会 | 移动到相邻节点、脱离封闭空间、触发低完成撤离 |
| 规则类 | 擦除、改写、隔离、复核规则 | 验证规则、擦除低级规则、隔离污染、改写规则 |
| 结算类 | 影响结算奖励、惩罚、评级或死亡判定 | 复核奖励、复核惩罚、重算评级 |
| 材料类 | 锻造、晋升、兑换或副本特殊交互材料 | 锻材、核心、凭证、兑换物 |
| 彩蛋类 | 神器、玩梗、测试玩具、隐藏奖励 | 万能口袋、测试道具、封印唯一效果 |

## 物品形态

`item_type` 用来告诉后端这件东西怎么进入背包、怎么装备、怎么扣耐久。

| 物品形态 | 说明 | 示例 |
| --- | --- | --- |
| `consumable` 消耗品 | 用完扣次数或移除 | 绷带、白蜡烛、结算复核券 |
| `weapon` 武器 | 占主武器/副手槽，可攻击、破坏、威慑或格挡 | 马桶搋子、拖把、消防斧、折叠椅 |
| `armor` 防具 | 占防具槽，提供减伤、抗污染、身份保护等持续效果 | 不锈钢盆、塑料雨衣、校服外套、防护服 |
| `accessory` 饰品 | 占饰品槽，提供护符、徽章、戒指、标记物效果 | 半枚护符、工牌、旧钥匙扣 |
| `equippable_tool` 可装备工具 | 占副手或特殊槽，提供工具判定、探查、维修等持续效果 | 坏掉的手电、热水壶线、指南针 |
| `material` 材料 | 用于锻造、维修、晋升、兑换 | 锻材、异常残片、副本核心 |
| `special` 特殊物 | 副本绑定物、进化媒介、唯一装备或封印物 | 主神小票、进化媒介、唯一任务物 |

装备是总称，指会占槽位并走耐久的物品；具体落地时优先写成武器、防具、饰品、可装备工具或特殊物。

## 等级强度边界

道具效果必须先符合等级，再看分类和风格。名字可以疯，强度不能越级。

| 等级 | 强度定位 | 建议数值范围 | 必须限制 |
| --- | --- | --- | --- |
| D | 新手小工具、轻量消耗品、生活化临时解法 | HP/SAN +20-35；通用判定 +1；特定场景判定 +2；减伤 1-2；可用 1-3 次或耐久 3-8；价格 20-80 | 不能撤离、替死、回满、改写规则、复核结算、直接揭示关键真相 |
| C | 稳定低阶道具，可作为普通商店主力 | HP/SAN +40-70；通用判定 +2；特定场景判定 +3；减伤 3-5；可用 1-4 次或耐久 5-12；价格 80-200 | 不能跳过核心条件，不能处理 B+ 规则，不能无代价解除严重状态 |
| B | 中阶战术道具，能改变一段局势 | HP/SAN +80-120；通用判定 +3；特定场景判定 +4；减伤 6-10；可移除一个中等状态；价格 220-500 | 不能单独解决主线；不能保证撤离或复活；规则效果必须有代价 |
| A | 高阶稀有道具，允许一次重大效果 | HP/SAN +150 或在安全节点回满单项；抵消一次致命伤害；触发低完成撤离；改写低级规则；复核一次结算；价格 600-1200 | 必须一次性、封印、冷却、阶段限制、债务、威胁时钟、污染或结算折扣至少命中一项 |
| S | 传说/神器/彩蛋级，影响死亡、结算或高价值规则节点 | 独特效果，不用简单数值堆强；价格 1500-5000 | 必须有 `seal_rank`、冷却、债务、污染、未知标记、惩罚副本或强副作用；低阶持有时只能触发降级效果 |

分类细化：

| 分类 | D/C 强度 | B 强度 | A/S 强度 |
| --- | --- | --- | --- |
| 增益类 | 小回复、低级状态缓解、短时抗性 | 大回复、移除中等状态、短时强抗性 | 回满、严重状态解除或复活前置，只能在安全节点/结算使用或付出高代价 |
| 防护类 | 抵消轻微伤害、污染接触、低级误认 | 抵消一次 B 级以下精神暗示/中额伤害 | 替死、致命伤抵消、高级污染隔离，必须一次性且有代价 |
| 探查类 | 方向提示、危险预警、低风险扫描 | 定位异常源、识别分支风险、标记实体 | 揭示关键真相的一部分，不能直接给出完整通关答案 |
| 线索类 | 校验矛盾、保存普通证言、标记线索点 | 锁定因果节点、保护关键证言 | 恢复被改写记忆或补全关键规则，但必须扣 SAN/涨威胁 |
| 工具类 | 开锁、照明、攀爬、维修等小加值 | 改变路径、争取额外行动或重试 | 临时开辟通道或跨越高危障碍，必须消耗稀有资源/耐久 |
| 伪装类 | 降低暴露度、短时身份遮蔽 | 对抗中级身份系统，失败触发风险 | 高级身份重写或长期遮蔽，必须带污染/债务/封印 |
| 位移类 | 只能脱困、跨越小障碍、移动到相邻安全点 | 强行脱离追击或换到相邻节点 | 撤离、逃生、跨区域移动，默认 A/S 且按低完成或惩罚结算 |
| 规则类 | 只能提示、验证、标记规则 | 擦除/隔离低级规则，必须扣 SAN 或涨威胁 | 改写规则、复核判定或隔离高级规则，必须高代价 |
| 结算类 | 不建议 D/C 出现，最多记录凭证 | 小幅提高探索度或奖励 roll | 复核奖励/惩罚/评级，必须接受新结果或承担债务 |
| 材料类 | 锻造、维修、兑换低阶物 | 合成中阶装备或晋升材料 | 高阶锻造核心/进化材料，本身不直接产生强效果 |
| 彩蛋类 | 不建议 D/C 出现 | 测试/玩梗但不能破坏规则 | 可以离谱，但必须封印、限次、冷却或仅测试池可见 |

物品形态强度：

| 形态 | D/C 强度 | B 强度 | A/S 强度 |
| --- | --- | --- | --- |
| 武器 | 伤害/破坏 +1 到 +2，或特定目标 +2，必须有耐久 | 伤害/破坏 +3，或特定目标 +4，可带一个小特效 | 高伤害、特攻、规则伤害，必须有耐久消耗、维修成本或副作用 |
| 防具 | 减伤 1-5、低级抗性、身份/环境小加值，必须有耐久 | 减伤 6-10 或抵抗一次中级效果 | 抵抗高级异常、致命保护或强伪装，必须封印/高修理成本 |
| 饰品 | 小幅判定加值、抗性或标记保护 | 中等抗性、一次触发式保护或线索保存 | 复合抗性、替死前置、规则护符，必须有封印或代价 |
| 可装备工具 | 特定场景 +1 到 +3，耐久低 | 改变一段行动路线或提供重试 | 临时跨越高危障碍或规则干涉，必须有耐久重耗或稀有材料消耗 |

越级处理：

- 如果一个效果看起来像 A/S，即使名字很生活化，也必须定为 A/S。
- 如果想把强效果降到 B 或以下，必须改成局部、低成功率、强场景限定或只给线索，不允许保留原强度。
- 价格不能用来买强度越级。低价只能反映副作用、封印、场景限制或回收价值低。
- 同一件道具同时有多个强效果时，以最高强度效果定级，并额外加代价。

## 定价流程

DS 生成道具初稿时不生成价格。价格必须在完整列表产出后，由规则维护者统一审校并补齐，避免不同批次的经济尺度漂移。

流程：

1. DS 分批生成 `道具名字 / 等级 / 分类 / 作用效果`。
2. 规则维护者合并完整列表，先删重、改名、修正越级效果、用途分类和物品形态。
3. 按同等级横向比较强度，再补 `price`。
4. 只有补齐价格、确认 `shop_allowed` / `gacha_allowed` / `seal_rank` 后，才允许写入正式 `item_catalog`。

定价参考：

| 稀有度 | 最终定价区间 |
| --- | ---: |
| D | 20-80 |
| C | 80-200 |
| B | 220-500 |
| A | 600-1200 |
| S | 1500-5000 |

调价规则：

- 多次使用、带耐久、强保护、规则类、撤离类、结算类价格偏高。
- 有明显副作用、封印限制、场景限制或只能在 `hub/settlement` 使用，可以适当降价。
- 普通商店只使用已人工定价的 D/C 商品，偶尔上架已人工定价的 B 商品。
- `price = 0` 只能表示未定价草稿或不可交易测试物，不能进入正式商店轮转。

## 道具目录生成提示词

下面这段可以直接给 DS，用来分批生成道具目录。

```text
你要为一个无限流文字副本游戏生成道具目录。

输出格式必须是 Markdown 表格，只输出表格，不要解释。
表头固定为：
| 道具名字 | 等级 | 分类 | 作用效果 |

字段要求：
- 道具名字：可以生活化、荒诞、怪谈化，但不能重复。
- 等级：只能是 D/C/B/A/S。
- 分类：只能从这些中文一级分类中选择：增益类、防护类、探查类、线索类、工具类、伪装类、位移类、规则类、结算类、材料类、彩蛋类。不要把武器、防具、装备写进分类。
- 作用效果：必须能被后端规则系统结算，写清数值、次数、耐久、使用阶段、装备/使用限制、代价、失败后果或封印限制。不能写“看情况”“由 GM 决定”“产生神秘效果”这种不可执行描述。
- 作用效果第一句必须写“物品形态：消耗品/武器/防具/饰品/可装备工具/材料/特殊物；”。武器、防具、饰品、可装备工具还必须写槽位和耐久。
- 不要生成价格列，不要写价格建议。价格会在完整列表审校后统一补。

重要定义：
- 道具包含消耗品、武器、防具、饰品、可装备工具、材料和特殊物。
- 武器不一定是明确武器。比如马桶搋子、拖把、热水壶线、消防斧、折叠椅都可以作为武器，只要它们可被装备或临时用于攻击/破坏/威慑。
- 防具不一定是传统护甲。比如不锈钢盆、塑料雨衣、校服外套、工牌、口罩都可以作为防具或饰品，只要它们可被穿戴/持有并提供持续防护、伪装或判定加成。
- 防护类是用途分类，防具是物品形态。例：不锈钢盆可以是“分类：防护类；物品形态：防具”，马桶搋子可以是“分类：工具类；物品形态：武器”。
- 武器、防具、饰品、可装备工具必须在作用效果里写槽位和耐久，例如“物品形态：防具；槽位 armor；耐久 8”。
- 武器的分类按主要用途决定，不写“武器类”：破坏/威慑写工具类，格挡/保护写防护类，异常污染/规则干涉写规则类，侦测实体写探查类。
- 武器必须写使用限制。常见限制包括：`str_min` 力量门槛、`agi_min` 敏捷门槛、`int_min` 智力门槛、`spi_min` 精神力基础门槛、`spi_current_min` 当前精神力门槛、`san_current_min` 当前 SAN 门槛、`rank_min` 阶位门槛、使用场景、禁止状态。
- 污染武器、邪神器官、怪谈武器必须写使用代价。常见代价包括：SAN 损失、污染增加、威胁时钟推进、债务、耐久重耗、冷却。
- 例如“邪神的心脏”应写成：分类 规则类；物品形态 武器/特殊物；槽位 special；限制 spi_min/spi_current_min/san_current_min/rank_min；使用后 SAN -35、污染 +12、威胁时钟 +1、冷却 3 轮。
- 普通消耗品必须写“一次性”或“可用 N 次”。

平衡规则：
- 效果强度必须符合等级。先判断道具实际效果属于 D/C/B/A/S 哪一级，再填写等级；不能把 A/S 效果伪装成 D/C/B 道具。
- D/C 以增益、探查、防护、线索、工具、普通武器/防具/饰品/可装备工具为主，效果小但有用。
- B 可以有战术价值，但不能直接跳过核心通关条件。
- A/S 是强道具，必须有代价、封印、冷却、使用阶段限制或失败后果。
- 规则类道具必须写清影响范围、使用阶段、次数、代价和失败后果；D/C 规则类只能提示/验证/标记，B 才能擦除/隔离低级规则，A/S 才能改写或复核高价值规则节点。
- 位移、撤离、替死、回满 HP/SAN、结算复核、规则改写默认属于 A/S 效果，必须有代价。
- 普通商店主要售卖 D/C，偶尔 B；A/S 不进普通商店。

等级强度边界：
- D：HP/SAN +20-35；通用判定 +1；特定场景判定 +2；减伤 1-2；可用 1-3 次或耐久 3-8；禁止撤离、替死、回满、改写规则、复核结算、直接揭示关键真相。
- C：HP/SAN +40-70；通用判定 +2；特定场景判定 +3；减伤 3-5；可用 1-4 次或耐久 5-12；禁止跳过核心条件、处理 B+ 规则或无代价解除严重状态。
- B：HP/SAN +80-120；通用判定 +3；特定场景判定 +4；减伤 6-10；可以移除一个中等状态或擦除/隔离低级规则，但必须有代价；不能单独解决主线。
- A：允许一次重大效果，如安全节点回满单项、抵消一次致命伤害、低完成撤离、低级规则改写、结算复核；必须一次性或有封印/冷却/债务/污染/威胁时钟/结算折扣。
- S：传说或神器级，必须有 seal_rank、冷却、债务、污染、未知标记、惩罚副本或强副作用；低阶持有时只能触发降级效果。
- 如果一个低等级道具看似强，必须缩小范围、降低成功率、增加场景限定或提升等级，不能只靠低价格平衡。

风格比例：
- 至少一半道具来自便利店、学校、出租屋、医院、老小区、办公室、地下车库等生活场景。
- 约三分之一道具要带怪谈味：普通外观，但被副本污染过。
- 少量彩蛋类可以离谱，但必须封印或有强限制。

生成数量：
- 本批生成 {数量} 个 {等级、分类或物品形态范围} 道具。
- 不要重复已有道具：{已有道具列表}。
```

使用建议：

- 只生成表格创意时，一批 50-80 个。
- 要求同时补后端 JSON 字段时，一批 20-40 个。
- 最稳的方式是按等级分批：D 40-50、C 35-45、B 25-35、A 12-20、S 5-10。

## 道具风格

道具可以分三层：

| 层级 | 来源/风格 | 规则定位 |
| --- | --- | --- |
| 普通道具 | 便利店、学校、出租屋、医院、老小区里能捡到的生活物 | 效果小、有梗、可商店/副本拾取 |
| 异常道具 | 看起来普通，但被副本污染过，有一点规则味 | 有触发条件、轻微代价或场景限定 |
| 神器/彩蛋 | 离谱、玩梗、测试玩具或隐藏奖励 | 不进普通商店；来自隐藏奖励、极低概率抽卡、测试池 |

示例方向：

- 普通道具：马桶搋子、半截粉笔、过期学生证、塑料雨衣、坏掉的手电、热水壶线、便利店小票。
- 异常道具：只会指向厕所的指南针、写了别人名字的创可贴、不会响的闹钟、能擦掉一句话的橡皮、越戴越像路人的口罩。
- 神器/彩蛋：内部内容包可以直球玩梗，例如“哆啦a梦口袋”；公开发版时换成影射名，例如“蓝色次元口袋”。

核心规则：

```text
名字可以疯，效果不能疯到系统不可判定。
一半实用，一半怪，一小撮离谱但封印。
```

## 道具使用与报废

道具目录不额外增加字段；一次性、多次使用、报废和轻量耐久都写进 `effect`。

| 形态 | 规则 | 示例 |
| --- | --- | --- |
| 一次性 | 使用后移出背包 | 绷带、白蜡烛、替身纸人、结算复核券 |
| 多次使用 | `effect` 写明剩余次数；每次成功触发后扣次数 | 安全绳、坏掉的手电、杂音收音机 |
| 场景消耗 | 只在特定场景扣次数；无效使用不扣或少扣 | 只会指向厕所的指南针、旧铜钥匙 |
| 报废保留 | 次数或耐久归 0 后变成 `broken` | 坏掉的手电、热水壶线 |
| 装备耐久 | 武器、防具、饰品、特殊装备统一走装备耐久规则 | 武器、防具、护符、盾、可装备工具 |

普通道具报废规则：

- `effect.uses = 1` 表示一次性。
- `effect.uses > 1` 表示可使用多次；每次成功触发后 `uses - 1`。
- `effect.durability` 表示轻量耐久；普通使用 -1，高风险使用 -2 到 -3，强行抵消后果 -5 以上。
- `uses = 0` 或 `durability = 0` 时，道具进入 `broken` 或直接移除，具体写在 `effect.on_depleted`。
- 普通道具默认不能维修，除非 `effect.repair_allowed = true`；可维修时维修价格不得低于原参考价的 30%。
- 武器、防具、饰品、可装备工具不使用普通道具次数规则，统一使用装备耐久系统。

示例：

```json
{
  "name": "马桶搋子",
  "rarity": "D",
  "category": "工具类",
  "item_type": "weapon",
  "equip_slot": "main_weapon",
  "effect": {
    "type": "multi_modifier",
    "weapon": { "scope": "melee_break", "physical_bonus": 1 },
    "tool": { "match_tags": ["blocked", "pipe", "drain", "toilet"], "dc_bonus": 2 },
    "side_effect": { "type": "social_disguise_delta", "amount": -1, "duration_rounds": 1 },
    "durability": 6,
    "on_depleted": "broken"
  }
}
```

## 规则类道具

规则类道具必须写清使用限制和使用代价，不能只写“改写/擦除/复核规则”。

规则类 `effect` 至少包含：

| 项目 | 要求 |
| --- | --- |
| 影响范围 | 只能影响低级规则、中级片段、指定标签规则或一次结算节点 |
| 使用阶段 | 写明只能在副本内、安全节点、结算阶段或 hub 使用 |
| 使用次数 | 一次性或固定次数；A/S 可加副本冷却 |
| 使用代价 | SAN、污染、威胁时钟、债务、稀有材料、装备损坏、奖励折扣至少选一种 |
| 失败后果 | 失败时的 SAN 损失、威胁时钟增加、污染增加或道具损坏 |
| 封印限制 | 高阶效果必须写 `seal_rank` 或低阶封印效果 |

规则类强度边界：

| 等级 | 允许效果 | 必须代价 |
| --- | --- | --- |
| D/C | 只能提示、验证、标记规则线索，不允许改写规则 | 消耗道具或少量 SAN |
| B | 可擦除/隔离低级规则，或验证中级规则片段 | SAN -10 到 -20，或威胁时钟 +1 |
| A | 可改写低级规则，或短暂隔离中级规则片段 | SAN -20 到 -35，威胁时钟 +2，或污染 +10 |
| S | 可复核/干涉一次高价值规则节点，但不能无代价通关 | 债务、污染、强制惩罚副本、奖励折扣或长冷却 |

示例：

```json
{
  "name": "弱规则改写笔",
  "rarity": "A",
  "category": "规则类",
  "effect": {
    "type": "rule_rewrite",
    "scope": "low_rule_only",
    "uses": 1,
    "cost": { "san_delta": -25, "threat_clock_delta": 2 },
    "failure": { "san_delta": -15, "item_break": true },
    "sealed_below": "A"
  }
}
```

## 可装备物

武器、防具、饰品和可装备工具是主要积分消耗口之一。默认规则里，可装备物不直接决定剧情胜负，但会影响战斗、破坏、威慑、防护、抵抗异常和特定行动判定。

可装备物类型：

| 类型 | 适用场景 | 默认加成方向 |
| --- | --- | --- |
| `melee` 近战 | 近身战斗、破坏、威慑 | HP 伤害、格挡 |
| `ranged` 远程 | 风险距离外攻击、压制 | HP 伤害、先手 |
| `arcane` 异常 | 规则、污染、怪谈实体 | SAN/规则干涉 |
| `tool` 工具 | 开锁、切割、攀爬、维修 | DC 加值、降低风险 |
| `armor` 防具 | 护甲、防护服、屏障、不锈钢盆 | 减伤、抗污染 |
| `accessory` 饰品 | 护符、徽章、工牌、戒指 | 小幅抗性、伪装、特殊触发 |

## 装备限制与污染武器

武器和强可装备物必须支持使用限制。限制由后端判定，GM 只能得到“可用/不可用/低配生效/代价已扣”的结果。

限制字段：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `rank_min` | 最低阶位 | `B` 才能完整使用 |
| `str_min` | 力量门槛 | `str >= 16` 才能挥动消防斧 |
| `con_min` | 体力门槛 | `con >= 14` 才能连续使用重装备 |
| `agi_min` | 敏捷门槛 | `agi >= 16` 才能使用高速/远程武器 |
| `int_min` | 智力门槛 | `int >= 18` 才能理解机关/术式武器 |
| `spi_min` | 精神力基础门槛 | `spi >= 18` 才能掌控规则/污染武器 |
| `spi_current_min` | 当前精神力门槛 | 当前 `spi_current >= 18` 才能承受规则/污染武器 |
| `san_current_min` | 当前 SAN 门槛 | 当前 SAN >= 80 才能拔出污染武器 |
| `evolution_tags_any` | 需要任一进化标签 | 需要 `beast`、`pollution_resist` |
| `ability_ids_any` | 需要任一能力 | 需要 `rule_probe` 或 `mental_anchor` |
| `forbidden_conditions` | 禁止状态 | `污染`、`失控` 时不能使用 |
| `scene_tags_any` | 使用场景限制 | 只能在梦境、镜面、医院类场景用 |

限制不足时：

| 情况 | 后端处理 |
| --- | --- |
| 未达 `rank_min` | 可持有但封印，不能装备或只触发低配效果 |
| 未达 `str_min` | 可以尝试，但行动 DC +2，失败耐久额外 -2；重武器默认不可连续使用 |
| 未达 `con_min` | 可以装备但不能连续高强度使用，强行使用会扣 HP 或增加疲劳状态 |
| 未达 `agi_min` | 可以使用但先手/命中惩罚 -2，高速武器可能拒绝触发特效 |
| 未达 `int_min` | 机关/术式武器只能触发基础效果，不能触发复杂特效 |
| 未达 `spi_min` | 规则/污染武器不可装备或只能封印持有 |
| 未达 `spi_current_min` | 规则/污染武器当前不可用，或使用后 SAN 代价翻倍 |
| 当前 SAN 低于 `san_current_min` | 直接拒绝使用；污染武器可改为触发反噬 |
| 带有禁止状态 | 不能使用，或触发失败后果 |

稀有度门槛建议：

| 稀有度 | 常规武器门槛 | 异常/污染武器门槛 |
| --- | --- | --- |
| D | 无门槛或 `str_min 10` | `spi_min 8`，`spi_current_min 6`，SAN 代价 0-5 |
| C | `str_min 12-14` | `spi_min 10-12`，`spi_current_min 8-10`，SAN 代价 5-10 |
| B | `str_min 16-20` | `spi_min 14-18`，`spi_current_min 12-15`，`san_current_min 60-90`，SAN 代价 10-20 |
| A | `str_min 24-30`，`rank_min A` | `spi_min 22-28`，`spi_current_min 16-22`，`san_current_min 100-140`，SAN 代价 20-35 或污染 +5-10 |
| S | `str_min 34+`，`rank_min S` 或封印段 | `spi_min 32+`，`spi_current_min 24+`，`san_current_min 150+`，SAN 代价 35+、污染、债务或惩罚副本 |

使用代价：

| 代价 | 说明 |
| --- | --- |
| `san_delta` | 使用后扣 SAN，适合污染、恐惧、规则干涉武器 |
| `hp_delta` | 使用者受伤、献血、透支体能 |
| `pollution_delta` | 增加污染值或污染标记 |
| `threat_clock_delta` | 威胁时钟推进 |
| `debt_delta` | 死亡债务或主神债务增加 |
| `durability_delta` | 额外耐久消耗，强触发必须扣更多 |
| `cooldown_rounds` / `cooldown_instances` | 冷却轮数或副本数 |

污染武器规则：

- 污染武器通常是 `item_type = weapon`，`category = 规则类`；如果主要用途是防护或探查，也可以是 `防护类` 或 `探查类`。
- 污染武器必须有至少一种限制：`spi_min`、`spi_current_min`、`san_current_min`、`rank_min`、`cooldown`、污染代价或封印。
- 污染武器每次使用必须有代价：SAN 损失、污染增加、威胁时钟推进、债务、耐久重耗至少一项。
- 污染武器扣 SAN 后，后端必须立刻扣减当前精神力 `spi_current`；连续使用会因为当前精神力下降而越来越难满足门槛。
- A/S 污染武器不能作为普通商店商品；只能来自抽卡、隐藏奖励、特殊商店或副本唯一掉落。
- 当使用者 SAN 低于门槛时，不允许 GM 自行判定“勉强成功”；后端要返回拒绝、反噬或低配效果。

示例：

```json
{
  "name": "邪神的心脏",
  "rarity": "S",
  "category": "规则类",
  "item_type": "weapon",
  "equip_slot": "special",
  "requirements": {
    "rank_min": "S",
    "spi_min": 32,
    "spi_current_min": 24,
    "san_current_min": 140,
    "forbidden_conditions": ["失控"]
  },
  "use_cost": {
    "san_delta": -35,
    "pollution_delta": 12,
    "threat_clock_delta": 1,
    "durability_delta": -8,
    "cooldown_rounds": 3
  },
  "effect": {
    "type": "rule_weapon",
    "scope": "entity_or_rule_node",

  }
}
```

## 进化系统

进化系统替代原“血统”。它不是单纯抽一个血统挂在身上，而是玩家身体强化到一个阶段时，选择自己的强化方向：

- 继续保持人类方向：稳定、低副作用、适合长期成长。
- 向其他生物或异常方向进化：强度更高、更有特性，但会带来污染、外貌、社交、SAN、债务或副本代价。

默认每名玩家有：

- 1 条主进化路径。
- 每个阶段 1 次进化选择。
- 最多 2 个副进化特质，通常来自低阶模板、奖励或隐藏副本。

进化阶段：

| 阶段 | 默认加成 | 选择内容 |
| --- | --- | --- |
| D | HP 或 SAN +10 | 基础适应：人类稳定 / 异化萌芽 |
| C | HP 或 SAN +20 | 一项轻被动或每副本 1 次小能力 |
| B | HP 或 SAN +40 | 抗性、形态变化或环境适应 |
| A | HP 或 SAN +80 | 强规则干涉或高代价形态 |
| S | HP 或 SAN +130 | 唯一形态，必须有副作用或冷却 |

进化成本：

| 升级 | 积分 | 进化碎片 | 等级/阶位限制 |
| --- | ---: | ---: | --- |
| 凡人 -> D | 200 | 30 | Lv1 |
| D -> C | 500 | 90 | Lv3 / C 阶 |
| C -> B | 1200 | 300 | Lv6 / B 阶 |
| B -> A | 2800 | 900 | Lv10 / A 阶 |
| A -> S | 6500 | 3000 | Lv15 / S 阶 |

进化方向：

| 方向 | 说明 | 平衡方式 |
| --- | --- | --- |
| 人类稳定 | 维持人类身体，强化体能、精神、学习和适应力 | 加成稳定，副作用少，强度峰值较低 |
| 夜行/阴影 | 夜视、潜行、黑暗适应 | 强依赖环境，光照/神圣/暴露场景受限 |
| 再生/兽化 | 自愈、爆发、物理抗性 | 社交伪装下降，污染或 SAN 代价 |
| 机械/义体 | 机关、科技、维修、抗毒 | 对电磁/锈蚀/规则怪谈弱点明显 |
| 污染抗性 | 抗精神污染，能直视异常 | 易累积污染标记，可能触发净化副本 |
| 镜界/门径 | 位移、脱困、空间交互 | 冷却长，失败时损坏装备或增加债务 |

人类方向不是“弱选项”：

- 人类稳定路线获得更高的通用判定、低污染、低债务风险。
- 人类稳定路线更容易触发“低污染通关”“无异化通关”等成就。
- 非人路线获得更强的场景特性，但必须承担可记录的副作用。

封印规则：

- 抽卡或隐藏奖励可以提前获得高阶进化模板。
- 阶位不足时，该模板进入 `sealed_evolution`。
- 封印模板只提供当前阶位允许的低阶效果，不能越级生效。
- S 级进化模板默认是封印体。

示例进化模板：

| 模板 | 阶段 | 效果 |
| --- | --- | --- |
| 人类稳定 | D | `con` 或 `int` +1，污染获得 -5% |
| 人类稳定 | B | 任意通用判定 +2，每副本 1 次抵消轻微状态 |
| 夜视体质 | D | 黑暗观察惩罚 -1 |
| 夜行者 | B | 黑暗视为普通环境；被黑暗实体伏击时风险 -1 级 |
| 再生体质 | D | 每副本结束后 HP +20 |
| 再生体质 | B | 每轮高压场景结束 HP +5，最多触发 5 次 |
| 机械亲和 | C | 科技、机械、机关判定 +2 |
| 半妖化 | B | HP +40；physical 伤害 -3；精神污染 +5% |
| 旧神污染抗性 | B | SAN +40；污染伤害 -5 |
| 镜界行者 | A | 每副本 1 次穿越镜面到相邻节点，SAN -25 |
| 门之眷属 | S | 每 3 副本 1 次打开非出口之门，必须接受等价代价 |
