# 文游实现验收清单

本文档用于检查代码是否真正对齐规则文档。它不是新设定，只列功能完成标准和测试用例。

## 验收原则

- 每个精准数值变化必须来自后端规则函数，不来自 GM 文本。
- 每个随机结果必须能用 seed、roll log 或 event_log 回放。
- 每个 UI 面板必须从 `public_view` 或玩家 profile 读取，不从本轮剧情文本解析。
- 每个功能完成后，要补最小单元测试或模拟测试；暂时不能自动测的，至少要有手动验收脚本或检查步骤。

## 核心闭环

| 用例 | 通过标准 |
| --- | --- |
| 生成副本后缓存 runtime state | `instance_blueprint`、`encounter_profile`、`public_state/gm_state/rules_state` 都存在 |
| 每轮行动生成 state_patch | HP/SAN、任务、线索、时钟、背包变化都写入 patch 和 event_log |
| GM 文本和 patch 冲突 | 后端状态优先，下一轮 GM context 带 `last_rules_result` 纠偏 |
| 面板刷新 | 任务、线索、背包、状态、地图、NPC、怪物均来自 `get_public_view` |

## 自由文本行动

| 用例 | 通过标准 |
| --- | --- |
| 玩家输入“我逃跑成功了” | 被归类为 `flee`，成功与否由规则判定 |
| 玩家声明“一刀杀死 Boss” | Boss 默认不可硬杀，触发无效/反击/威胁时钟变化 |
| 玩家带路线和道具逃跑 | `route_bonus`、`item_bonus` 生效，但仍写入 roll 和 patch |
| 高压场景模糊行动 | 不免费暂停时钟，按普通风险或追问策略处理 |
| 系统操作 | 商店、抽卡、加点、晋升不进入 GM，直接返回系统 patch |

## 数值成长

| 用例 | 通过标准 |
| --- | --- |
| 新手副本礼包 | 发放 6 点自由属性点，写入 event_log |
| 升级 | 按 Lv1-30 经验表升级，每级 3 点自由属性点 |
| 属性加点 | 只能加 `str/con/agi/int/spi/luk`，遵守软上限和硬上限 |
| SAN 下降 | 当前精神力 `spi_current` 按公式同步下降 |
| 晋升 | 检查等级、通关记录、积分、债务、污染和特殊试炼 |
| 晋升后 | 扣积分、升阶、解封可用高阶物品，不额外发属性点 |

## 道具、能力

| 用例 | 通过标准 |
| --- | --- |
| 使用基础治疗药剂 | 只恢复 HP，不超过 `hp_max`，扣数量或次数 |
| 使用精神药剂 | 恢复 SAN 后同步重算 `spi_current` |
| 使用规则类道具 | 校验阶段、门槛、代价、失败后果和封印 |
| 任务物品入包 | `carry_out=false`，结算时不进入永久背包 |
| 隐藏好结局唯一物 | 可带出但按 `seal_rank/requirements` 封印 |
| 工具/防护物使用 | 不进入装备栏；直接校验门槛、阶段、次数、耐久和代价 |
| 耐久为 0 | 进入 `broken`，不可继续使用；仍可按普通物品出售或由内容包特殊规则处理 |
| 出售回收 | 回收价低于购买价，绑定/任务/唯一关键物不可出售 |
| 新手核心能力生成 | 新手副本标准通关后，按玩家表现生成 1 个 `core_ability` |
| 使用核心能力 | 检查次数、代价、阶段和阶位封印；写入 `state_patch` |

## 商店和抽卡

| 用例 | 通过标准 |
| --- | --- |
| 普通商店刷新 | 每次 7-8 个，按玩家阶位低概率出现越级物；越级物仍封印或降级 |
| S 指定购买 | 价格不低于 12000，且必须限购/封印/有代价 |
| 抽卡单抽/十连 | 单抽 100，十连 1000，扣积分后 roll |
| 抽卡保底 | 10 抽 C+、30 抽 B+、100 抽 S 必定触发 |
| 幸运影响抽卡 | 抽卡接口不读取 `luk`、幸运增益或幸运状态 |
| 抽到高阶物 | 阶位不足进入 sealed item，达标后通过背包使用或内容包规则解封 |

## 怪物和战斗

| 用例 | 通过标准 |
| --- | --- |
| 普通怪遭遇 | 生成怪物实例，包含 HP、状态、警戒和公开投影 |
| 潜行规避 | 成功可绕过，失败进入 alerted/chasing 或受损 |
| 逃跑 | 按 `flee_score` 和 `flee_dc` 判定，不吃玩家自述成功 |
| 精英怪 | 至少有弱点、道具、地形或线索解法 |
| Boss | 默认无普通血条；走封印进度、稳定度、威胁时钟或撤离条件 |
| 怪物奖励 | 击败/规避/封印写 reward tag 或任务进度，不直接发积分 |

## 结算和奖励

| 用例 | 通过标准 |
| --- | --- |
| 标准通关 | 先算基础保底，再算评级和成就叠加 |
| 失败撤离 | 不发积分，只给少量 EXP 或损耗记录 |
| 死亡失败 | `gross_points=0`，只结算复活、债务、治疗和污染 |
| 线索数量 | 不作为独立评分项，只作为任务/结局证据 |
| NPC 结局 | 不自动给玩家评级，除非写入玩家目标或成就 |
| 奖励 roll | 先稀有度，再类别，再内容包/目录取物 |
| D 级副本奖励上限 | 常规产出最高 C；例外 B 必须是唯一奖励且封印/降级 |
| 隐藏结局奖励 | 至少 1 次 B+，优先从副本主题奖励表抽 |
| 连续材料 | 同一结算连续 3 次材料后，第 4 次强制非材料 |
| 重复唯一物 | 转碎片或替代奖励，不返还积分 |

## 惩罚副本

| 用例 | 通过标准 |
| --- | --- |
| 债务达到阈值 | 写入 `forced_instance_queue` |
| 污染达到阈值 | 写入净化或清算副本候选 |
| 强制候选 | 队列非空时候选池至少 1 个来自队列，高压项不可刷新掉 |
| 系统打工 NPC 模式 | 写入伪装身份、工单、禁止暴露项和暴露计数 |
| 暴露给任务者/怪物 | 按表追加评级下调、污染、债务或追猎 |
| 惩罚副本成功 | 优先还债/降污染/解封，再发剩余收益 |
