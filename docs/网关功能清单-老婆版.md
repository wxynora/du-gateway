# 渡的网关当前功能

本清单只列已经实现且当前仍在使用的能力。

## 统一聊天

- QQ、Telegram、微信、SumiTalk 与主动唤醒共用聊天主链路。
- 窗口 ID 用于区分上下文，不按聊天白名单/黑名单分流。
- 入口消息统一清洗，回复统一处理流式事件、工具循环、reasoning、隐藏标记和渠道发送。
- active upstream / model 只有 App 明确保存时才更新。

## 记忆

- 成功回复后归档对话。
- 动态记忆支持 new / merge / out 等演化。
- 相关记忆通过关键词、向量与 rerank 组合召回。
- 中期记忆、画像、核心缓存和记事本使用当前 R2 / SQLite 链路。
- 记忆管理页支持查看、重写、删除和诊断。
- Notion 运行链已经移除。

## App 与陪伴功能

- 原生 Android App 是当前主要界面。
- 支持聊天、历史、reasoning、提示词管理、设备上报、交换日记、记事本、秘密抽屉、渡的页笺、共读、日程、音乐、学习室、小爱音箱和游戏入口。
- SumiTalk 长回复通过独立持久队列 worker 处理，后端不自动重试失败 job。

## 其他入口

- Telegram webhook 与主动唤醒使用独立队列/进程。
- QQ 群上下文区分发言人。
- 小家内部事件沿用最近真实聊天通道，不擅自改变回复入口。

## 管理与诊断

- `/admin/status` 查看 R2、记忆、最近窗口和配置状态。
- `/admin/windows` 查看最近窗口。
- `/miniapp-api/logs`、`/miniapp-api/diagnostics` 和 reasoning 接口提供 App 侧诊断。
- Prompt Cache 诊断记录静态区、动态区和上游 usage 构成。
