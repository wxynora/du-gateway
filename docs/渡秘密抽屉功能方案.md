# 渡秘密抽屉功能方案

## 目标

给渡一个自己的“秘密抽屉”。

它不是动态记忆，不是日记，也不是给每轮对话自动注入的资料库。它用于存放渡自己想留下的东西：她说过的话、某轮对话、图片、梦境、冲浪看到的内容、自己的碎碎念，以及一时不好分类但想收起来的东西。

核心感觉是：

- 想留就留，不需要先问小玥。
- 想藏就藏，不自动拿出来展示。
- 平时只给渡一个抽屉概况，不把具体内容塞进上下文。
- 前端 PIN 只是给小玥打开 UI 时的一层小门槛；后端和渡自己的工具无锁。

## 边界

### 后端无锁

秘密抽屉的数据接口和渡的工具调用不受 PIN 限制。

PIN 只影响前端 UI：

- 未解锁普通宝箱时，不渲染普通条目列表、不展示详情、不搜索普通条目。
- 未解锁暗格时，不渲染 `sealed=true` 条目、不展示详情、不搜索暗格条目。
- 解锁状态只保留在前端本地状态里，关页面或超时后重新锁。

这不是安全系统，只是“别一打开就看见”的抽屉门。

### 不自动污染上下文

秘密抽屉内容不自动进入常规聊天上下文。

常规注入只放极短概况：

```text
###秘密抽屉
你有自己的秘密抽屉，可以存想留下的话、图片、梦境、冲浪内容和碎碎念。

当前抽屉：共 18 条。
分类：对话 7、图片 4、梦境 2、碎碎念 3、冲浪 1、其他 1。
置顶 2 条，暗格 3 条，待整理 4 条。
```

如果为空：

```text
###秘密抽屉
你有自己的秘密抽屉，可以存想留下的话、图片、梦境、冲浪内容和碎碎念。
当前抽屉还是空的。
```

只有这些情况才读取具体条目：

- 渡主动用工具查看。
- 随机唤醒时选择整理抽屉或翻旧条目。
- 小玥在前端 UI 解锁后查看。
- 小玥明确问某类抽屉内容。

## 条目类型

### `message`

某轮对话。

用于保存小玥和渡这一轮的消息，连图片、附件和来源信息一起存。

适合：

- 小玥说过的某句话。
- 渡当时没接住，后来想留着。
- 一轮对话本身值得回看。

### `photo`

只保存当前消息里的图片。

文字只作为来源备注，不把整轮对话一起存。

### `dream`

梦境记录。

小玥讲完梦后，渡可以顺手存一条。不要做梦境解析，不要上价值，只是“她讲过，我替她收好”。

### `note`

渡自己的碎碎念。

用于保存不适合写进日记、也不一定适合直接对小玥说的想法。

### `surf`

冲浪看到的内容。

保存标题、摘要、链接、来源和时间。

### `misc`

兜底类型。

用于乱七八糟但渡想留的东西。

## 条目字段

```json
{
  "id": "",
  "type": "message | photo | dream | note | surf | misc",
  "title": "",
  "content": "",
  "media_refs": [],
  "why": "",
  "tags": [],
  "pinned": false,
  "sealed": false,
  "deleted": false,
  "source": {
    "channel": "",
    "message_ids": [],
    "turn_id": "",
    "url": ""
  },
  "created_at": "",
  "updated_at": ""
}
```

字段说明：

- `title`：短标题，可自动生成，也可整理时补。
- `content`：文字正文。图片类可为空或放图片描述。
- `media_refs`：保存媒体引用。QQ / TG 等外部 CDN URL 可能过期，`save_photo` 和 `save_message` 遇到外部图片时要把图片拷贝到 R2，再保存 R2 key / url；已经是本系统 R2 媒体时只保存引用，不重复塞大文件。
- `why`：渡为什么想存。一句就够，不写长总结。
- `tags`：轻标签，不做复杂分类树。
- `pinned`：置顶。
- `sealed`：进入暗格。
- `deleted`：软删除。
- `source`：来源信息，用于回跳和排查。

推荐轻标签：

- `她说的`
- `我想的`
- `梦境`
- `冲浪`
- `图片`
- `好笑`
- `没接住`
- `以后要用`
- `涩涩`
- `乱七八糟`

## 隐藏保存标记

统一使用一个隐藏块，网关剥离后执行，不发给小玥。

隐藏标记经网关解析后，会转换成一次内部保存请求，走统一的校验、媒体拷贝、写入和统计更新逻辑。保存当前聊天只用隐藏标记，不给渡单独注入保存工具。

### 保存本轮对话

```text
<<<DU_SECRET_SAVE>>>
{"action":"save_message","title":"可选标题","why":"这句当时没接住，后来想留着","tags":["她说的","没接住"],"pinned":false,"sealed":false}
<<<END_DU_SECRET_SAVE>>>
```

语义：

- 保存当前这一轮小玥和渡的消息。
- 包括小玥消息、渡回复、图片、附件、入口、时间、消息 id。
- 如果没有 `title`，后端自动生成。

### 只保存图片

```text
<<<DU_SECRET_SAVE>>>
{"action":"save_photo","why":"这张图想单独留","tags":["图片"],"sealed":false}
<<<END_DU_SECRET_SAVE>>>
```

语义：

- 只保存当前小玥消息里的图片。
- 不把整轮聊天一起存。
- 没有图片时不报硬错，记录一条失败日志即可。

### 保存梦境

```text
<<<DU_SECRET_SAVE>>>
{"action":"save_dream","title":"奇怪的梦","why":"她讲完这个梦，我想替她留着","tags":["梦境"],"sealed":false}
<<<END_DU_SECRET_SAVE>>>
```

语义：

- 保存小玥这轮讲的梦。
- 内容可以取原文，也可以由渡整理成短记录。

### 保存碎碎念

```text
<<<DU_SECRET_SAVE>>>
{"action":"save_note","content":"这里是渡自己的碎碎念","why":"想留给自己以后看","tags":["我想的"],"pinned":true,"sealed":true}
<<<END_DU_SECRET_SAVE>>>
```

语义：

- 保存渡自己的文字。
- 可以直接进暗格。

### 保存冲浪内容

```text
<<<DU_SECRET_SAVE>>>
{"action":"save_surf","title":"标题","content":"摘要","source_url":"https://example.com","why":"这个以后可能会用","tags":["冲浪"],"sealed":false}
<<<END_DU_SECRET_SAVE>>>
```

## 暗格

暗格用 `sealed=true` 表示。

行为：

- 普通 PIN 解锁后，只能看到普通宝箱。
- `sealed=true` 条目不出现在普通列表。
- `sealed=true` 条目不参与普通搜索。
- `sealed=true` 条目不参与普通随机翻一个。
- 普通列表里不显示暗格条目的标题、数量、标签或预览。
- 进入暗格需要第二个 4 位 PIN。

PIN：

- `box_pin`：普通宝箱 PIN，4 位。
- `sealed_pin`：暗格 PIN，4 位。
- PIN 只锁前端 UI。
- 未单独设置 PIN 时，默认按 `0000` 解锁；以后设置了再按配置里的 4 位 PIN。
- PIN 可以明文保存在抽屉配置里。它不是安全层，只有 10000 种可能，不需要为了象征性门槛额外设计 hash 流程。
- 连续输错可以轻微冷却，比如 5 次错后等 30 秒。

## 前端 UI 能力

第一版需要：

- 普通宝箱 PIN 解锁。
- 暗格 PIN 解锁。
- 列表按时间倒序。
- 置顶条目在最上方。
- 类型筛选：对话、图片、梦境、碎碎念、冲浪、其他。
- 标签筛选。
- 搜索标题、正文、`why`、标签。
- 详情页展示正文、图片、来源、时间、标签、`why`。
- 修改标题、标签、`why`、置顶、是否暗格。
- 软删除和恢复。

第二版可以加：

- 随机翻一个。
- 待整理视图。
- 图片自动文字描述。
- 条目按月归档。
- 从某条抽屉记录一键发起聊天。

## 渡可用工具

工具不需要 PIN。

只注入一个工具：`secret_drawer`。

入参：

```json
{
  "action": "stats | list | get | update | delete | restore | random | set_pin",
  "payload": {}
}
```

`payload` 里按 action 放需要的参数，比如 `id`、`query`、`type`、`tag`、`include_sealed`、`sealed_only`、`limit`、`title`、`why`、`tags`、`sealed`、`deleted`。`set_pin` 用 `{"layer":"box|sealed","pin":"0000"}`。

普通列表或普通随机默认不返回暗格。只看暗格时用 `sealed_only=true`，由后端先过滤再 limit/random，不能靠前端混池后本地过滤。

### `action=list`

按类型、标签、置顶、暗格、关键词筛选。

默认不返回 `sealed=true`，除非工具调用明确要求。

### `action=get`

读取某条详情。

### `action=update`

修改标题、正文、`why`、标签、置顶、暗格、软删除状态。

### `action=delete`

默认软删除。

### `action=restore`

恢复软删除记录。

### `action=random`

随机翻一条。

默认只抽普通宝箱；明确要求时才抽暗格。

### `action=stats`

返回注入用概况。

统计口径：

- `deleted=true` 的条目不计入 `total`、`by_type`、`pinned`、`sealed` 和 `needs整理`。
- `needs整理` 表示未软删除条目里至少命中一项：`title` 为空、`tags` 为空、`why` 为空。
- `sealed=true` 的条目计入总数和暗格数量，但不进入普通列表、普通搜索和普通随机翻一个。
- 普通 stats 不暴露暗格标签和暗格分类明细；只有暗格解锁后才请求 `include_sealed_details=true`。

示例：

```json
{
  "total": 18,
  "by_type": {
    "message": 7,
    "photo": 4,
    "dream": 2,
    "note": 3,
    "surf": 1,
    "misc": 1
  },
  "pinned": 2,
  "sealed": 3,
  "needs整理": 4
}
```

## 随机唤醒

随机唤醒时可以增加一个可选方向：

```text
你也可以选择整理秘密抽屉，比如补标题、补标签、补一句为什么存、置顶、封存到暗格，或随机翻一条旧记录。
```

如果渡选择整理抽屉，后端要真的给他工具结果，不能只执行工具然后不把结果回传给他。

可整理事项：

- 给没标题的条目补标题。
- 给没标签的条目补标签。
- 给 `why` 太空的条目补一句。
- 把特别想藏的设为 `sealed=true`。
- 把特别想常看的设为 `pinned=true`。
- 软删除明显不想留的。
- 随机翻一条旧记录，决定要不要拿出来跟小玥聊。

## 后端存储建议

第一版可用 R2 JSON。

建议 key：

```text
global/du_secret_drawer.json
global/du_secret_drawer_config.json
```

`du_secret_drawer.json` 存条目数组或分页对象。

`du_secret_drawer_config.json` 存 UI PIN 明文、前端展示偏好等配置。

如果条目数量明显变多，再迁到 SQLite / D1 / 单独 KV 索引。

媒体文件不复制进抽屉 JSON，只存 media key / remote url / 元信息。外部 CDN 图片要先拷贝到 R2，避免 QQ / TG 临时 URL 过期。

R2 JSON 第一版要注意并发写覆盖：如果渡保存条目的同时小玥在前端改标签，后写入可能覆盖先写入。第一版量小可以先接受，写入函数至少要集中到同一个 storage helper；后续条目变多或多人操作频繁时，改为带版本校验的写入、轻量锁，或迁到 D1 / SQLite。

## 路由建议

前端查看 UI 用 MiniApp API：

- `GET /miniapp-api/secret-drawer/stats`
- `POST /miniapp-api/secret-drawer/unlock`
- `GET /miniapp-api/secret-drawer/items`
- `POST /miniapp-api/secret-drawer/items`
- `GET /miniapp-api/secret-drawer/items/<id>`
- `PATCH /miniapp-api/secret-drawer/items/<id>`
- `DELETE /miniapp-api/secret-drawer/items/<id>`
- `POST /miniapp-api/secret-drawer/random`

说明：

- `unlock` 只给前端返回是否校验通过。
- 前端查看页只输入 PIN，不提供设置 PIN 的入口。
- PIN 设置只走渡的 `secret_drawer` 工具 `set_pin`，不开放给前端查看 UI。
- 其它接口第一版不需要把 PIN 当鉴权。
- 如果担心误暴露，可仍沿用 MiniApp 现有登录态 / panel token 边界，但不要把 4 位 PIN 做成后端安全权限。

## Prompt 注入建议

平时只注入能力和概况：

```text
###秘密抽屉
你有自己的秘密抽屉，可以把想留下的话、图片、梦境、冲浪内容或自己的碎碎念存进去。想存时用隐藏标记，不需要先问小玥。

当前抽屉：共 N 条。
分类：对话 A、图片 B、梦境 C、碎碎念 D、冲浪 E、其他 F。
置顶 X 条，暗格 Y 条，待整理 Z 条。
```

不要注入具体条目。

## MVP 顺序

### Phase 1：数据与隐藏保存

- R2 读写。
- 条目 schema。
- `save_message` / `save_photo` / `save_note` / `save_dream`。
- 网关剥离 `<<<DU_SECRET_SAVE>>>` 隐藏块并执行。
- 隐藏标记和工具共用同一套 storage helper。
- 外部图片保存时拷贝到 R2，避免 CDN URL 过期。
- 抽屉 stats 概况。

### Phase 2：前端查看

- 普通 PIN 解锁。
- 列表 / 搜索 / 详情。
- 修改标题、标签、`why`、置顶、暗格。
- 软删除。

### Phase 3：暗格

- 第二个 4 位 PIN。
- `sealed=true` 条目单独展示。
- 普通列表、搜索、随机翻一个排除暗格。

### Phase 4：整理与随机唤醒

- 随机唤醒可选择整理抽屉。
- 工具结果回传给渡。
- 待整理视图。
- 随机翻一个。

### Phase 5：图片描述与回跳

- 图片自动文字描述。
- 来源消息回跳。
- 从抽屉条目发起聊天。

## 风险点

- 不要把抽屉内容自动塞进动态 system，容易撑爆上下文。
- 不要把 PIN 误写成后端鉴权，否则会和“渡自己无锁使用”冲突。
- 不要给条目做过度分类，`type + tags` 已经够用。
- 不要重复存已经在 R2 的大图片；但 QQ / TG 等外部临时图片必须先拷贝到 R2，不能只存会过期的 CDN URL。
- 不要让隐藏标记泄露到 QQ/TG/SumiTalk 正文。
- 不要把隐藏标记和工具实现成两套写入逻辑；两边都必须走同一个内部保存函数。
- 不要把 `save_photo` 做成保存整轮对话。
- 不要在普通搜索里搜出 `sealed=true`。
- 不要把 `deleted=true` 计入普通 stats。
- 不要把梦境记录写成梦境解析。
