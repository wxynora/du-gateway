# 功能自检清单

跑一次 `python scripts/call_test.py` 会做下面这些检查。

## 脚本里已覆盖的自检

| 项 | 说明 |
|----|------|
| 1. GET /health | 网关存活 |
| 2. GET /v1/models | 模型列表代理（用第一个模型名，不瞎编） |
| 3. POST /v1/chat/completions | 聊天转发 + 白名单管道（注入、存档、动态层 DS） |
| 4. GET /admin/windows | 最近窗口列表，含 whitelisted/blacklisted |
| 5. GET /admin/whitelist | 白名单列表 |
| 6. GET /admin/blacklist | 黑名单列表 |

以上通过即说明：转发、模型列表、白名单/黑名单、管理端接口都可用；R2 可写（存档会写；NoSuchKey 为桶里尚无该 key，属正常）。

## 未做自动化、需部署/人工或按需测的

| 功能 | 建议怎么验 |
|------|------------|
| 黑名单窗口响应末尾「（黑名单）」 | 用黑名单 window_id 发一条聊天，看回复末尾是否有该提示 |
| 新窗 + user 含「测试」→ 进黑名单 | 新开一个窗口，第一条消息发「测试」，再发一条看是否只转发且带（黑名单） |
| 时间注入（今日/周几/时段、😭x分钟后老婆终于回我了） | 看 system 注入内容或对话表现 |
| 多目标 fallback（TARGET_AI_URLS） | 配多个 URL，把第一个关掉或返回非 2xx，看是否自动用下一个 |
| 每 4 轮 DeepSeek 总结 | 白名单内同一窗口连续聊 4 轮，看 R2 global/summary.txt 是否更新（需 DEEPSEEK_API_KEY） |
| 小本本（📝小本本更新） | 对话里触发一次，看 R2 notebook、Notion 是否有写入 |
| 卧室通道（tag 卧室） | 触发一次卧室内容，看 Notion 卧室页面是否有追加 |
| 核心缓存 sync_to_notion / sync_from_notion | 管理端调接口，看 Notion 数据库是否一致 |
| RikkaHub 接网关 | 在 RikkaHub 填网关地址，实际对话 + 选模型，确认正常 |
