# 功能自检清单

当前没有通用 `scripts/call_test.py`。每次按改动范围做定向验证。

## 基础检查

```bash
.venv/bin/python -m py_compile app.py routes/chat.py pipeline/pipeline.py
.venv/bin/python -c "import app"
git diff --check
```

## 只读运行检查

| 接口 | 用途 |
| --- | --- |
| `GET /health` | 主网关存活 |
| `GET /v1/models` | 当前模型列表可读 |
| `GET /admin/windows` | 最近窗口可读 |
| `GET /admin/status` | R2、记忆和配置状态可读 |

## 按改动范围验证

- 聊天：用真实入口发一条，核对回复通道、流式事件和归档日志。
- SumiTalk：同时检查主网关和独立 chat worker。
- Telegram：分别检查 webhook worker 或主动唤醒 worker。
- 记忆：使用 mock / 临时存储验证，不写生产 R2。
- App：验证真实原生 APK 行为，不用 MiniApp 页面代替原生回归。

不存在聊天白名单/黑名单自检，也不存在 Notion 同步自检。
