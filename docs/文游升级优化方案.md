# 文游升级优化方案

更新时间：2026-05-16

## 一句话目标

把文游从 Telegram 群指令式玩法，升级成一个完整的 App 内无限流文字游戏：

- 用户可以在 App 里选择副本、随机进入、继续当前副本、查看历史归档。
- 游玩时可以随时查看任务、背包、角色状态、线索、地点、关系和行动历史。
- 主体验保持文字叙事，但 UI 承担“游戏面板”的职责：展示状态、承接操作、降低重复生成成本。
- 低变化、可结构化的信息由 App 固定展示，不让每次查看任务/背包都重新写一遍。

这次升级的目标不是把文游做成一堆后台表单，而是做成“主神空间 + 副本行动界面”的移动端文字游戏。

## 体验基调

### 关键词

- 无限流
- 主神空间
- 副本档案
- 生存任务
- 倒计时
- 线索收集
- 道具使用
- 低声压迫感

### 视觉方向

- 深色底，骨白文字，暗红作为危险/系统强调色。
- 少量扫描线、网格、编号、系统角标，制造“副本载入”的感觉。
- 主剧情阅读区必须干净，不能被过多卡片切碎。
- 信息面板可以有游戏感，但不做花哨装饰堆叠。

### 交互原则

1. 主线行动永远在主界面完成。
2. 任务、背包、状态、线索都用抽屉或轻二级页承接，不打断剧情。
3. 常用动作一键可点，自由行动可以手写。
4. 任何发送/使用/重开类动作都要防重复点击。
5. 副本状态可恢复，刷新 App 后用户不应丢失当前位置。
6. UI 能展示的固定信息，就不要靠剧情区重复讲。

## 一级信息架构

文游入口保持在 MiniApp 内，内部形成自己的四层结构：

```text
文游首页 / 主神空间
  -> 副本选择 / 随机匹配 / 当前副本
    -> 副本主界面
      -> 任务 / 背包 / 状态 / 线索 / 地点 / 关系 / 历史
        -> 结算 / 归档 / 复盘
```

### 页面清单

1. 文游首页：主神空间
2. 副本选择页
3. 随机副本设置弹层
4. 副本详情页
5. 入场动画
6. 副本主界面
7. 任务抽屉
8. 背包抽屉
9. 道具使用确认层
10. 角色状态面板
11. 线索板
12. 地点/地图面板
13. 人物关系面板
14. 行动历史页
15. 副本结算页
16. 副本归档页
17. 文游设置页

## 1. 文游首页：主神空间

首页不是普通功能入口，而是“进入副本前的主神空间”。

### 页面内容

顶部展示：

- 文游标题，例如 `无限副本`
- 当前存活状态摘要
- 最近一次副本记录

核心区域展示：

- 当前副本卡
- 继续副本按钮
- 选择副本按钮
- 随机进入按钮
- 历史归档入口

底部可以展示：

- 最近解锁的副本
- 最近获得的称号/结局
- 最近死亡记录

### 当前副本卡

有进行中副本时显示：

- 副本名
- 副本编号
- 类型
- 难度
- 当前阶段
- 当前任务摘要
- 生命/理智/污染等核心状态
- 最近一次行动时间

没有进行中副本时显示空状态：

```text
当前没有正在运行的副本。
选择一个副本，或者把命交给随机。
```

### 交互

- 点 `继续副本`：进入副本主界面。
- 点 `选择副本`：进入副本选择页。
- 点 `随机进入`：打开随机副本设置弹层。
- 点 `历史归档`：进入副本归档页。
- 点当前副本卡：进入副本主界面。
- 长按当前副本卡：打开更多操作，例如重开、放弃、查看摘要。

## 2. 副本选择页

用于主动选择副本。

### 页面结构

顶部：

- 返回按钮
- 标题：`选择副本`
- 搜索入口

筛选区：

- 类型筛选
- 难度筛选
- 长度筛选
- 状态筛选

副本列表：

- 推荐副本
- 已解锁副本
- 最近玩过
- 收藏副本

### 类型筛选

建议初始类型：

- 全部
- 规则怪谈
- 都市异闻
- 校园
- 医院
- 古宅
- 末日
- 科幻
- 民俗
- 密室
- 高压生存
- 推理解谜

### 难度筛选

- 新手
- 普通
- 困难
- 噩梦
- 不明

### 长度筛选

- 短篇：适合十几轮内结束
- 标准：常规副本
- 长篇：多阶段副本
- 连续副本：副本链/系列任务

### 副本卡片字段

每张副本卡显示：

- 副本名
- 副本编号
- 类型标签
- 难度标签
- 预计长度
- 简短氛围描述
- 初始危险提示
- 解锁状态
- 是否有历史结局

示例：

```text
雾港倒计时
规则怪谈 / 难度 B / 标准
午夜之后，港口的广播会叫出不该存在的名字。
```

### 交互

- 点卡片：展开副本详情。
- 点 `进入`：进入入场动画。
- 点收藏：加入收藏。
- 搜索时实时筛选本地列表。
- 已锁定副本显示锁图标和解锁条件。
- 最近玩过的副本显示结局状态，例如存活、死亡、逃脱、未完成。

## 3. 随机副本设置弹层

从首页点击 `随机进入` 打开。

### 弹层内容

用户可以配置：

- 难度偏好
- 类型偏好
- 长度偏好
- 风格偏好
- 禁用标签

### 难度偏好

- 随机
- 新手
- 普通
- 困难
- 噩梦

### 类型偏好

用多选标签：

- 规则怪谈
- 校园
- 医院
- 末日
- 古宅
- 科幻
- 民俗
- 都市异闻

### 风格偏好

- 生存
- 悬疑
- 推理
- 探索
- 高压
- 慢热
- 多人物
- 少人物

### 禁用标签

用于让用户排除不想玩的内容，例如：

- 过度血腥
- 密集追逐
- 复杂数学
- 太长
- 太压抑

### 交互

- 用户可以不选，默认全随机。
- 点 `开始随机` 后显示短暂匹配动画。
- 匹配动画结束后进入入场动画。
- 点遮罩或取消按钮关闭弹层。

## 4. 副本详情页

从副本列表展开或进入单独详情页。

### 展示内容

- 副本名
- 副本编号
- 类型、难度、长度
- 开场简介
- 初始任务
- 危险提示
- 推荐玩法
- 历史结局
- 解锁奖励

### 操作

- 进入副本
- 收藏/取消收藏
- 查看历史记录
- 重新挑战

### 交互细节

- 简介不宜太长，默认展示 3-5 行。
- 危险提示用短句，不剧透核心机制。
- 如果该副本正在进行，主按钮变成 `继续副本`。
- 如果该副本曾经失败，显示 `重新挑战`。

## 5. 入场动画

当前已有基础方向，后续作为标准副本进入动效。

### 画面元素

- 黑场
- 扫描线或网格
- 系统角标：`MAIN GOD SYSTEM`
- 副本编号
- `欢迎来到 {副本名}`
- `努力生存下去吧。`
- 类型、难度标签

### 交互

- 点击任意位置跳过。
- 动画结束自动进入副本主界面。
- 减少动态模式下只显示静态入场页。

### 使用时机

- 新副本开始。
- 随机副本匹配完成。
- 副本阶段大切换时可以使用弱化版，例如 `第二幕已开启`。
- 不用于普通行动返回，避免打断节奏。

## 6. 副本主界面

这是文游核心界面。

### 页面布局

```text
顶部状态栏
剧情阅读区
系统事件浮层
底部行动区
快捷工具栏
```

### 顶部状态栏

显示：

- 返回按钮
- 副本名
- 当前阶段
- 倒计时/回合数
- 角色状态入口
- 更多菜单

顶部状态栏应尽量克制，不堆满数值。详细状态放进状态面板。

### 剧情阅读区

内容类型：

- 剧情正文
- 用户行动
- 行动结果
- 系统提示
- 任务更新
- 物品获得/失去
- 状态变化

不同消息需要视觉区分：

- 剧情正文：普通文本块
- 用户行动：靠右或有轻微行动标识
- 系统提示：窄条/角标
- 任务更新：带任务图标和短提示
- 物品变化：带物品图标/徽章
- 状态变化：短浮层或嵌入提示

### 底部行动区

包含：

- 多行输入框
- 发送按钮
- 快捷行动按钮
- 工具按钮

快捷行动建议：

- 观察
- 检查
- 交谈
- 移动
- 使用物品
- 等待

### 输入交互

- 输入框默认一行，输入变长后最多展开到 4 行。
- 聚焦输入框后，快捷行动收起成一行图标。
- 发送中禁用重复提交。
- 发送失败保留输入内容。
- 剧情返回后自动滚动到最新。
- 用户手动上滑查看历史时，不强制滚到底部，显示 `回到最新` 按钮。

### 更多菜单

右上角更多菜单包含：

- 查看任务
- 查看背包
- 查看状态
- 查看线索
- 查看地点
- 查看关系
- 行动历史
- 重新开始副本
- 放弃副本

危险操作需要二次确认。

## 7. 任务系统 UI

任务不是单独页面优先，而是主界面的底部抽屉。

### 入口

- 主界面快捷按钮
- 顶部状态栏红点
- 任务更新提示条

### 面板结构

分区：

- 主线任务
- 支线任务
- 隐藏任务
- 已完成
- 已失败

### 任务卡字段

- 任务标题
- 当前目标
- 进度
- 状态
- 失败条件
- 剩余时间/回合
- 相关地点
- 相关人物

### 任务状态

- 未解锁
- 进行中
- 可提交
- 已完成
- 已失败
- 已隐藏

### 交互

- 点任务卡展开详情。
- 已完成和已失败默认折叠。
- 有任务更新时入口出现红点。
- 打开任务面板后红点消失。
- 任务可关联线索、地点、人物；点击关联项跳到对应面板。

### 显示示例

```text
主线任务：活到天亮
当前目标：找到能避开广播巡逻的房间
进度：2 / 4 条线索
失败条件：午夜后连续三次回应广播
```

## 8. 背包系统 UI

背包用于展示和使用物品。

### 入口

- 主界面快捷按钮
- `使用物品` 快捷行动
- 物品获得提示

### 分类

- 全部
- 道具
- 线索
- 装备
- 消耗品
- 任务物品

### 物品卡字段

- 图标或文字徽章
- 名称
- 数量
- 品质/危险等级
- 简短描述
- 可用状态

### 物品详情

点开物品后显示：

- 名称
- 完整描述
- 获得来源
- 当前数量
- 当前是否可用
- 可能用途
- 使用按钮
- 丢弃/标记按钮

### 使用物品流程

1. 用户点物品。
2. 展开详情。
3. 点 `使用`。
4. 出现使用确认层。
5. 用户可以填写“想怎么用”。
6. 点 `确认使用`。
7. 关闭确认层。
8. 主剧情区出现行动记录。

### 使用确认层

内容：

- 物品名
- 使用对象选择
- 场景提示
- 补充输入框
- 确认使用
- 取消

对象选择可以先做成简单按钮：

- 对自己使用
- 对当前场景使用
- 对人物使用
- 自定义描述

### 不可用状态

不可用时不隐藏物品，而是显示原因：

- 当前场景不可使用
- 缺少目标
- 已损坏
- 需要满足任务条件
- 数量不足

## 9. 角色状态 UI

角色状态用于替代剧情里反复描述“你现在怎么样”。

### 入口

- 顶部状态按钮
- 更多菜单
- 状态变化提示

### 核心状态

- 生命
- 理智
- 体力
- 污染
- 饥饿/口渴
- 伤势
- 异常状态
- 临时增益
- 临时减益

### 展示形式

- 核心数值用条形进度。
- 异常状态用标签。
- 严重状态置顶。
- 正面状态和负面状态分区。

### 状态变化提示

主界面短提示：

```text
理智 -5
获得状态：轻微恐惧
伤势加重：左臂撕裂
污染 +1
```

### 交互

- 点状态项展开说明。
- 严重负面状态可以显示应对建议。
- 状态变化提示自动消失，但可在行动历史里查到。

线索板承接调查、推理、解谜。

### 入口

- 主界面工具按钮
- 任务详情关联项
- 获得线索提示

### 分类

- 全部
- 人物
- 地点
- 事件
- 物品
- 规则
- 未分类

### 线索卡字段

- 线索标题
- 分类
- 发现地点
- 发现时间/回合
- 简短摘要
- 标记状态

### 标记状态

用户可以给线索打标：

- 关键
- 可疑
- 已使用
- 待验证

### 交互

- 点线索展开详情。
- 支持搜索。
- 支持按分类筛选。
- 支持只看关键线索。
- 可从线索跳转到相关任务、地点、人物。

### 空状态

```text
还没有可用线索。
先活过开场。
```

## 11. 地点 / 地图面板

文游不一定需要真实地图，但需要地点结构，帮助用户知道自己在哪、能去哪。

### 入口

- 主界面快捷按钮
- `移动` 快捷行动
- 更多菜单

### 展示内容

- 当前地点
- 可移动地点
- 已探索地点
- 未探索地点
- 危险地点
- 被锁定地点

### 地点卡字段

- 地点名
- 状态：当前 / 可前往 / 锁定 / 危险 / 已封锁
- 简短描述
- 关联任务
- 关联线索

### 交互

- 点可前往地点，弹出移动确认。
- 点当前地点，显示详情。
- 锁定地点显示解锁提示。
- 危险地点移动前二次确认。

### 地图表现

第一版不必做复杂平面图，可以做成“地点列表 + 当前地点突出”。后续可以升级成节点地图。

## 12. 人物关系面板

无限流副本里 NPC 和队友很重要，需要单独面板承接。

### 入口

- 主界面更多菜单
- `交谈` 快捷行动
- 剧情中人物名点击

### 展示内容

- 已知人物
- 队友
- 敌对对象
- 失踪/死亡人物
- 不明身份人物

### 人物卡字段

- 名称
- 身份
- 当前状态
- 信任度/危险度
- 最近互动
- 关联任务
- 关联线索

### 交互

- 点人物展开详情。
- 可以选择 `交谈`、`询问线索`、`使用物品`。
- 人物死亡/失踪后状态明显变化。
- 不明身份人物用隐藏信息展示，例如 `???`。

## 13. 行动历史

### 入口

- 更多菜单
- 结算页
- 归档页

### 展示内容

- 用户行动
- 剧情结果摘要
- 任务变化
- 物品变化
- 状态变化
- 线索获得

### 筛选

- 全部
- 我的行动
- 系统事件
- 任务变化
- 物品变化

### 交互

## 15. 副本归档页

- 进行中

- 死亡
- 放弃
- 收藏

### 归档卡字段

- 副本名
- 结局

## 16. 文游设置页

文游自己的偏好设置，不和全局设置混在一起。

### 可配置项

- 默认副本难度
- 默认随机类型偏好
- 是否显示入场动画
- 是否自动展开任务更新
- 是否使用紧凑剧情排版
- 是否减少动态效果
- 行动确认策略
- 危险操作二次确认

### 行动确认策略

- 仅危险行动确认
- 使用物品时确认
- 所有行动都不确认

## 17. 快捷行动设计

快捷行动是文游从“纯输入框”升级成 App 的关键。

### 初始快捷行动

- 观察：查看当前场景。
- 检查：检查指定物品/地点。
- 交谈：与当前人物互动。
- 移动：打开地点面板。
- 使用物品：打开背包并进入使用模式。
- 等待：推进时间。

### 快捷行动交互

- 点击快捷行动可以直接生成预填文本。
- 例如点 `观察`，输入框出现 `观察周围环境`。
- 用户可以直接发送，也可以编辑。
- 点 `使用物品` 不直接发送，而是打开背包选择物品。

### 长按快捷行动

后续可支持：

- 展开二级选项
- 自定义快捷短语
- 查看动作说明

## 18. 前端状态与提示

### Loading

- 首页加载：轻骨架。
- 副本列表加载：卡片骨架。
- 随机匹配：全屏短动画。
- 行动发送：底部发送按钮 loading。
- 面板加载：面板内部骨架，不遮挡主剧情。

### Empty

- 无当前副本：引导开始。
- 背包为空：`背包空空，至少目前还没被塞进奇怪东西。`
- 任务为空：`当前还没有明确任务。`
- 线索为空：`还没有可用线索。`
- 历史为空：`还没有值得回头看的记录。`

### Error

- 行动失败：toast + 保留输入。
- 面板加载失败：面板内重试按钮。
- 使用物品失败：物品详情内提示原因。
- 副本载入失败：回到首页并给重试入口。

### Red Dot

需要红点的入口：

- 任务更新
- 获得新物品
- 状态异常变化
- 新线索
- 新地点
- 人物关系变化

红点规则：

- 只提示“有变化”，不长时间闪烁。
- 打开对应面板后清除。
- 严重状态可以保留警示色。

## 19. 移动端细节

### 输入区

- 固定在底部。
- 避免被键盘遮挡。
- 输入聚焦时，工具栏收起。
- 发送按钮保持可触达。

### 抽屉

任务、背包、状态、线索优先使用底部抽屉。

抽屉要求：

- 点击遮罩关闭。
- 下滑关闭。
- 上滑展开到近全屏。
- 顶部有拖拽柄。
- 内容滚动不影响背景剧情区。

### 安全区

- 适配 iPhone 底部安全区。
- 顶部状态栏不贴边。
- 按钮高度不低于 44px。

### 文本阅读

- 剧情正文行高充足。
- 长段落自动分段。
- 系统提示不抢正文层级。
- 不使用过小字号展示关键状态。

## 20. 动效规范

### 强动效

只用于：

- 入场动画
- 副本阶段切换
- 结算出现

### 中动效

用于：

- 抽屉展开
- 获得物品
- 任务更新
- 状态变化

### 弱动效

用于：

- 按钮点击反馈
- 标签切换
- 卡片展开

### 不应该动的地方

- 剧情正文不要频繁跳动。
- 背包列表不要每次刷新重排闪烁。
- 任务面板不要用复杂弹跳动画。

### 减少动态

开启减少动态后：

- 关闭扫描线。
- 关闭闪烁。
- 抽屉只淡入淡出。
- 入场动画变为静态页。

## 22. 数据展示边界

这里说的是前端展示边界，不展开后端实现。

### 适合固定展示的信息

- 副本编号
- 类型
- 难度
- 当前阶段
- 当前任务
- 任务进度
- 背包物品
- 角色状态
- 线索列表
- 地点列表
- 人物关系
- 行动历史摘要

- 行动结果

- 重要物品变化

- 全部角色数值
- 全部线索详情
- 重复的副本规则说明

## 23. 完整版验收标准

### 首页

- 没有当前副本时，用户能清楚知道怎么开始。
- 有当前副本时，用户能一键继续。
- 随机和选择副本入口都明显。

### 副本选择

- 能按类型、难度、长度筛选。
- 能查看副本详情。
- 能进入指定副本。
- 能配置随机副本偏好。

### 主界面

- 能阅读剧情。
- 能输入自由行动。
- 能使用快捷行动。
- 能查看任务、背包、状态、线索。
- 行动发送中不会重复提交。
- 刷新后能恢复界面状态。

### 任务

- 能区分主线、支线、隐藏、完成、失败。
- 能看到当前目标和进度。
- 有更新提示。

### 背包

- 能分类查看物品。
- 能打开物品详情。
- 能完成使用确认流程。
- 不可用物品有原因。

### 状态

- 能看到核心状态和异常状态。
- 状态变化有轻提示。
- 严重状态足够醒目。

### 线索 / 地点 / 人物

- 线索可分类、搜索、标记。
- 地点能表达当前位置和可前往地点。
- 人物能表达身份、状态、关系变化。

### 结算 / 归档

- 副本结束后能看到结局。
- 能查看关键选择和完成度。
- 能重新挑战。
- 能回到主神空间。

## 24. 落地顺序建议

虽然目标形态要做齐，但实现可以分顺序，避免一次改到失控。

### 第一阶段：主体验完整闭环

- 文游首页
- 副本选择
- 随机副本弹层
- 入场动画
- 副本主界面
- 任务抽屉
- 背包抽屉
- 状态面板
- 行动发送防重复

### 第二阶段：游戏面板补齐

- 线索板
- 地点面板
- 人物关系面板
- 行动历史
- 红点和状态变化提示
- 道具使用确认流程完善

### 第三阶段：长期玩法

- 副本归档
- 结算页
- 重新挑战
- 收藏副本
- 解锁记录
- 文游设置页

### 第四阶段：体验打磨

- 副本阶段切换动效
- 更精细的空状态
- 紧凑/沉浸阅读模式
- 自定义快捷行动
- 线索关联任务/地点/人物

## 25. 不做成什么

为了不把文游做歪，明确几个不要：

- 不做成后台管理表格。
- 不把所有信息都塞进首页。
- 不在剧情区反复打印完整任务/背包。
- 不让每个小操作都弹大 modal。
- 不把任务、背包、状态做成三个割裂页面。
- 不让随机副本配置复杂到像填问卷。
- 不为了“游戏感”牺牲文字阅读。
