# 一键清空所有记录

`POST /admin/wipe_all` 会删除网关 R2 数据，并清空本地最近窗口、上次回复时间和 SQLite 运行数据。这是破坏性接口，只用于明确的环境重置。

## 调用

```bash
curl -X POST "你的网关地址/admin/wipe_all?confirm=wipe_all"
```

也可以把确认值放进 JSON body：

```json
{"confirm":"wipe_all"}
```

没有 `confirm=wipe_all` 时接口返回 400，不执行删除。

成功响应中的字段：

- `r2_deleted_keys`：删除的 R2 key 数量；
- `local_cleared_files`：清理的本地文件和 SQLite 表数量，数量会随当前表结构变化，不是固定值。

不会删除 `.env` 和 `data/emoji_mapping.json`。

未经辛玥当轮明确允许，不得在生产环境调用这个接口，也不得为了测试接口而调用。
