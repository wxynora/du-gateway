# 一键清空所有记录（傻瓜版）

测试完想清掉网关里的全部记录时用。**会删掉**：

- **R2 里**：所有对话存档、总结、动态层、核心缓存、小本本、图片描述
- **本机 data/**：白名单、黑名单、最近窗口列表、上次回复时间

**不会删**：`data/emoji_mapping.json`（表情包对照表，老婆可编辑）、`.env` 配置。

---

## 怎么清（三步）

### 1. 确认网关地址

- 本机跑网关：`http://localhost:5000` 或 `http://你的电脑IP:5000`
- 云服务器：`https://你的网关域名`

下面用「你的网关地址」代替，例如 `https://xxx.com`。

---

### 2. 用 Postman / Apifox 等（可选）

- 选 **POST**，URL 填：`你的网关地址/admin/wipe_all?confirm=wipe_all`
- 发送后看到 `{"ok":true,...}` 即成功。
- 或用下面「命令行」一步搞定。

---

### 3. 用命令行清（复制即用）

在 **PowerShell** 或 **命令提示符** 里（把 `https://你的网关域名` 换成你的网关地址）：

```powershell
curl -X POST "https://你的网关域名/admin/wipe_all?confirm=wipe_all"
```

**本机**示例：

```powershell
curl -X POST "http://localhost:5000/admin/wipe_all?confirm=wipe_all"
```

看到类似下面说明成功：

```json
{"ok":true,"r2_deleted_keys":123,"local_cleared_files":4}
```

- `r2_deleted_keys`：R2 里删了多少个 key（0 表示本来就没数据）
- `local_cleared_files`：本地清空了 4 个文件（白名单、黑名单、最近窗口、上次回复时间）

---

## 没带确认会怎样？

若忘记加 `confirm=wipe_all`，接口会返回 400，并提示你要带确认参数，**不会删任何东西**。

---

## 小结

| 方式       | 操作 |
|------------|--------|
| **命令行（推荐）** | `curl -X POST "你的网关地址/admin/wipe_all?confirm=wipe_all"` |
| Postman 等 | POST `你的网关地址/admin/wipe_all`，Query 参数 `confirm=wipe_all` |

说明：必须用 **POST** 或 **DELETE**，浏览器地址栏直接打开（GET）会报方法不允许。

清空后，所有对话/白名单/黑名单等都会重置，相当于全新环境。
